CQRS — Command Query Responsibility Segregation

Commands (writes) are routed to the Primary node; Queries (reads) go to the Replica. The router inspects the first SQL keyword to decide automatically.

                 Client Request

                       ▼

               │  CQRS Router  │ ◄── classify SQL keyword
               │  (this page)  │     INSERT/UPDATE/DELETE → command
               └───────┬───────┘     SELECT/…            → query
                       │
           ┌───────────┴───────────┐
           │                       │
     COMMAND path             QUERY path
    (write model)            (read model)
           │                       │

    │ UPDATE      │        │ aggregations    │
    │ DELETE      │        │ reports         │
