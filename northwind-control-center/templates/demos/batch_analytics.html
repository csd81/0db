▶ Batch Analytics SQL Server (row-store) vs pandas/NumPy (columnar) idle
Row-store (SQL Server) — stores one row contiguously. Fast for: INSERT/UPDATE, single-row SELECT by PK. Slow for: full-table scans, column aggregations (must read unused columns).
Columnar (BigQuery / pandas) — stores one column contiguously. Fast for: aggregations, GROUP BY, analytics. Cache-friendly for vectorised ops. Slow for: single-row point lookups (must scan all columns separately).
