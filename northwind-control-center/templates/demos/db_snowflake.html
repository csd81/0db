Snowflake

Cloud Data Warehouse · Separated compute/storage · Micro-partitions · Time Travel

Snowflake launched in 2012 with a radical idea: build a data warehouse that lives entirely in the cloud, with compute and storage separated so each can scale independently. Its micro-partition storage, virtual warehouses, and instant data sharing across organizations made it the fastest-growing database company in history — reaching a $70B valuation at IPO. Today it is the default enterprise data warehouse for companies running dbt pipelines and Fivetran data loads.

Micro-partitions & Clustering 
-- Standard table (auto micro-partitioned)
CREATE TABLE orders (
  order_id    NUMBER AUTOINCREMENT PRIMARY KEY,
  customer_id NUMBER,
  order_date  DATE,
  region      VARCHAR(50),
  total       NUMBER(12,2),
  payload     VARIANT  -- JSON/Avro/Parquet semi-structured
);

-- Explicit clustering key (improves partition pruning)
ALTER TABLE orders
  CLUSTER BY (region, TO_DATE(order_date, 'YYYY-MM'));

-- Zero-copy clone (instant, no data copied)
CREATE TABLE orders_backup CLONE orders;

-- Time Travel: query table as of yesterday
SELECT * FROM orders AT (OFFSET => -86400)
WHERE region = 'EMEA';
VARIANT (Semi-structured) 
-- Load JSON directly into VARIANT
INSERT INTO orders (payload)
SELECT PARSE_JSON('{
  "customer": {"name": "Alice", "tier": "gold"},
  "items": [{"sku":"A1","qty":2},{"sku":"B3","qty":1}],
  "shipping": {"method": "express", "days": 2}
}');

-- Query VARIANT with colon path syntax
SELECT
  payload:customer:name::STRING AS name,
  payload:shipping:method::STRING AS ship,
  ARRAY_SIZE(payload:items) AS item_count
FROM orders;

-- Lateral FLATTEN to unnest arrays
SELECT o.order_id, f.value:sku::STRING AS sku
FROM orders o,
LATERAL FLATTEN(input => o.payload:items) f;
QUALIFY & MATCH_RECOGNIZE 
-- QUALIFY: filter on window functions (no subquery!)
SELECT customer_id, total,
  RANK() OVER (PARTITION BY region ORDER BY total DESC) AS rnk
FROM orders
QUALIFY rnk <= 3;   -- top 3 per region, inline

-- PIVOT: rotate rows to columns
SELECT * FROM monthly_sales
PIVOT (SUM(amount) FOR month IN ('Jan','Feb','Mar','Apr'))
ORDER BY product;

-- MATCH_RECOGNIZE: pattern matching in sequences
-- Find sessions: page views followed by a purchase
SELECT *
FROM events
MATCH_RECOGNIZE (
  PARTITION BY user_id ORDER BY event_time
  MEASURES FIRST(view.url) AS landing_page
  PATTERN (view+ purchase)
  DEFINE
    view     AS event_type = 'pageview',
    purchase AS event_type = 'order'
);
Streams & Tasks (ELT) 
-- Stream: CDC on a Snowflake table
CREATE STREAM orders_stream ON TABLE orders;

-- Stream exposes new/changed rows with metadata
SELECT METADATA$ACTION,  -- INSERT/DELETE
       METADATA$ISUPDATE,
       order_id, total
FROM orders_stream;

-- Task: scheduled SQL (like a cron job)
CREATE TASK refresh_dashboard
  WAREHOUSE = 'XSMALL'
  SCHEDULE = '5 MINUTE'
AS
  MERGE INTO dashboard_summary d
  USING (SELECT region, SUM(total) AS revenue
         FROM orders_stream GROUP BY region) s
  ON d.region = s.region
  WHEN MATCHED THEN UPDATE SET d.revenue = s.revenue
  WHEN NOT MATCHED THEN INSERT VALUES (s.region, s.revenue);
Killer Use Cases
Enterprise DWH: Replace on-premises Teradata/Netezza. No capacity planning, pay per second of compute. dbt transforms, Fivetran/Airbyte loads, Tableau/Looker BI on top.
Data Sharing: A vendor shares a live dataset with 500 customers. Each customer queries live data — no ETL, no data copy. Snowflake Marketplace powers commercial data products.
ELT pipelines: Snowpipe ingests from S3, Tasks transform, Streams capture CDC. The entire pipeline lives inside Snowflake with no external orchestrator needed for simple cases.

Micro-partitions

Micro-partition: Tables are divided into 50–500 MB immutable columnar files stored in object storage. Each micro-partition stores metadata: min/max per column (zone maps). The query optimizer uses zone maps to skip irrelevant partitions without scanning them.

No indexes needed: Automatic partition pruning via zone maps replaces manual index creation. Add a clustering key if zone map pruning is insufficient (high cardinality columns, date range queries).

Result cache: Identical queries return the cached result in milliseconds without consuming VW compute credits. Cache is shared across all users of the account.

Virtual Warehouse Sizing 
-- Create and size a virtual warehouse
CREATE WAREHOUSE analytics_wh
  WAREHOUSE_SIZE = 'LARGE'    -- XS / S / M / L / XL / 2XL..
  AUTO_SUSPEND = 60           -- pause after 60s idle
  AUTO_RESUME = TRUE          -- wake on query
  MAX_CLUSTER_COUNT = 5       -- multi-cluster for concurrency
  MIN_CLUSTER_COUNT = 1;

-- Switch warehouse for a session
USE WAREHOUSE analytics_wh;

-- Monitor credit consumption
SELECT warehouse_name,
  SUM(credits_used) AS credits,
  SUM(execution_time_ms)/1000 AS exec_sec
FROM SNOWFLAKE.ACCOUNT_USAGE.WAREHOUSE_METERING_HISTORY
WHERE start_time >= DATEADD('day', -7, CURRENT_DATE)
GROUP BY warehouse_name ORDER BY credits DESC;
