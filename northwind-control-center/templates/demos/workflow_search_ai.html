🗄️ Storage & Data Layer
Redis
Prompt caching + conversation session state. TTL-keyed by session ID. Sub-millisecond GET/SET. Saves GPU cost by avoiding redundant inference on identical prompts.
Google Bigtable / Apache HBase
Stores raw web crawl data (billions of URLs). Column-family structure — each row is a URL, columns are crawl timestamps + content hashes. Scales to petabytes.
Google Spanner
Globally distributed relational DB for user accounts, billing, rate limits. True external consistency via TrueTime API (atomic clock + GPS). 99.999% SLA.
Pinecone / Milvus / FAISS
Vector databases — store high-dimensional embeddings (1536-dim for text-embedding-3-large). ANN (Approximate Nearest Neighbor) search with HNSW or IVF index for RAG retrieval in <50ms.
⚙️ Compute & Processing
Ray (Anyscale)
Distributed Python framework for both training and serving. ray.serve handles model deployment with autoscaling. Used by OpenAI for large-scale inference orchestration.
NVIDIA A100/H100 GPU Clusters
Tensor-parallel inference splits model weights across 8–16 GPUs. FP8/FP16 mixed precision. NVLink interconnect at 600 GB/s. PagedAttention (vLLM) for KV cache management.
Google TPU v4/v5
Google's custom ASIC — 275 TFLOPS of bfloat16 compute. 4096-chip "pods" for training trillion-parameter models. Used for Gemini training.
Apache Kafka / Google Pub/Sub
Event bus for query telemetry, safety signal propagation, and RLHF feedback pipeline. Topics partitioned by user_id for ordered processing.
🌐 Infrastructure & Protocols
Kubernetes (GKE / EKS)
Thousands of GPU pods managed by K8s. Custom schedulers pack GPU jobs onto nodes. HPA (Horizontal Pod Autoscaler) scales inference replicas on queue depth.
Server-Sent Events (SSE)
HTTP/1.1 streaming — server pushes token-by-token response. text/event-stream content type. Simpler than WebSocket for unidirectional streaming; works through CDN proxies.
Content-Based Routing
Envoy proxy routes requests based on model ID, user tier, and geographic region. Canary deployments route 1% of traffic to new model versions before full rollout.
Safety Infrastructure
Constitutional AI classifiers run in parallel to the main model. Llama Guard / OpenAI Moderation API filter harmful content. Latency budget: <10ms for safety check.
Workflow Deep Dive
1
Ingestion: API Gateway + Rate Limiting
Request hits Cloudflare or AWS API Gateway. JWT auth validated. Rate limits checked against Redis counter key rate:{user_id}:{minute}. Idempotency key stored to deduplicate retried requests. P99 latency here must be <5ms.
2
Safety Pre-processing & Tokenization
Input passed through a lightweight classifier (DistilBERT fine-tuned on CSAM/hate/violence). Prompt hashed with SHA-256 → check Redis prompt cache. If cache hit, skip inference entirely. Tokenizer (BPE or SentencePiece) converts text to integer token IDs.
3
RAG: Embedding → Vector Search
Query is encoded by an embedding model (e.g., text-embedding-3-large, 1536 dims) → nearest-neighbor search in Pinecone/Milvus using HNSW index. Returns top-K document chunks by cosine similarity. These become the "context" injected into the LLM prompt. Retrieval latency: ~20–50ms.
4
LLM Inference: Tensor Parallelism + KV Cache
Prompt (system + context + user query) dispatched to GPU cluster. Tensor parallelism splits attention heads across GPUs via NVLink. vLLM's PagedAttention manages KV cache in non-contiguous memory blocks — avoids fragmentation, enables 2–4× higher throughput. Each forward pass generates one token. A 100-token response = 100 forward passes (autoregressive decoding).
5
Streaming Delivery via SSE
Each generated token is immediately flushed to the HTTP response stream as data: {"token": "Hello"}\n\n. Client-side JS appends tokens to the DOM. Final token sends data: [DONE]. Response also cached in Redis for 24h (SHA-256 key) to serve identical queries without re-inference.
⚡ Why Not RDBMS for Vectors?
SQL B-tree indexes are optimized for exact lookups. ANN search over 100M 1536-dim vectors requires HNSW or IVF graph structures — impossible to replicate in PostgreSQL without pgvector extension (and even then it's slower than dedicated vector DBs).
🧠 RAG vs Fine-Tuning
RAG injects fresh context at inference time — no retraining needed. Fine-tuning bakes knowledge into weights but goes stale. Production systems use both: fine-tuned models for style/behavior, RAG for up-to-date factual retrieval.
💸 Token Economy
A single A100 can process ~20 tokens/second for a 70B model. At 100 requests/second, you need 5 A100s just for inference. Prompt caching (saving KV cache across sessions) cuts GPU cost by ~70% for repeated contexts. This is why Claude's pricing separates "cache write" vs "cache read" tokens.
🔒 CAP Position
AP system — prefers availability over strict consistency. A slightly stale vector index is acceptable. Redis prompt cache can miss (stale=serve, no cache). The only CP component is Spanner (billing/auth) where consistency is non-negotiable.
