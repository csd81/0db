How it works: The Central Office (left) cancels orders and updates addresses in real time. Meanwhile the Driver (right) marks packages delivered or failed — but is offline, so the two databases diverge. When signal returns, the Merge Agent (middle) syncs bidirectionally: driver confirmations are pushed up ↑, dispatcher changes are pulled down ↓. If both sides changed the same package, that is a conflict (⚡) — resolved automatically by the Central Office wins rule. This is eventual consistency: no node is blocked while offline, but all nodes converge to the same state after sync.
