SHRINK reference (use sparingly) 
-- Reclaim unused space, keeping 10% free
USE master
DBCC SHRINKDATABASE(northwind, 10)

-- ⚠ Avoid in production — causes index fragmentation.
-- Prefer: set initial file sizes correctly from the start.
DBCC CHECKDB — Live Integrity Check 
DBCC CHECKDB('northwind')
  WITH TABLERESULTS, NO_INFOMSGS, ALL_ERRORMSGS
-- Returns rows only if errors found; empty = ✓ clean
Page Allocation per Table 
SELECT tablename, totalpages, unusedPages, usedPages
FROM sys.indexes i
JOIN sys.partitions p ON ...
JOIN sys.allocation_units a ON ...
WHERE OBJECTPROPERTY(object_id, 'IsUserTable')=1
ORDER BY totalpages DESC
3 Error Types & Recovery
1. Connection lost
Log rolls back all uncommitted transactions automatically
2. Server crash
On restart: roll forward log → roll back uncommitted
3. Data file lost
Backup log → restore full + diff + log → rollback uncommitted

RESTORE DATABASE syntax reference — destructive, never run live 
USE [master]
RESTORE DATABASE [northwind]
FROM DISK = N'C:\...\Backup\nw'
WITH FILE = 1,
     NOUNLOAD,
     REPLACE,   -- overwrite existing DB
     STATS = 5  -- progress every 5%
BACKUP DATABASE syntax 
-- Full backup
BACKUP DATABASE northwind
TO DISK = N'C:\Backup\nw_full.bak'
WITH STATS = 10

-- Differential backup
BACKUP DATABASE northwind
TO DISK = N'C:\Backup\nw_diff.bak'
WITH DIFFERENTIAL, STATS = 10

-- Transaction log backup (FULL mode only)
BACKUP LOG northwind
TO DISK = N'C:\Backup\nw_log.bak'
WITH STATS = 10

ALTER ROLE db_datareader ADD MEMBER nw_user
Built-in Roles
Server Level
sysadmin serveradmin dbcreator public
Database Level — Positive
db_owner db_datareader db_datawriter
Database Level — Negative (DENY)
db_denydatareader db_denydatawriter
db_owner + db_denydatawriter: can SELECT & execute procs but INSERT/UPDATE fails — useful for read-heavy analytical roles.
Privilege Verbs 
-- Grant execute on one procedure
GRANT EXECUTE ON sp_list_employees TO nw_user

-- Grant execute on ALL objects in DB
GRANT EXECUTE TO nw_user

-- Schema-level grant (preferred — survives new objects)
GRANT SELECT, UPDATE ON SCHEMA::Sales TO my_role

-- Remove explicit grant (inherited may still apply)
REVOKE EXECUTE ON sp_list_employees FROM nw_user

-- Hard block — overrides any GRANT
DENY INSERT ON Orders TO nw_user
Live Demo — Stored Procedure with GRANT 
CREATE PROCEDURE sp_c12_list_employees
    @city varchar(50) = 'London'
AS
    SELECT EmployeeID, LastName, FirstName, Title, City, Country
    FROM Employees WHERE City = @city
-- Then: GRANT EXECUTE ON sp_c12_list_employees TO nw_user
Role Membership — Live Query 
SELECT DP1.name AS DatabaseRoleName,
       ISNULL(DP2.name,'No members') AS DatabaseUserName
FROM sys.database_role_members DRM
    RIGHT JOIN sys.database_principals DP1 ON ...
    LEFT  JOIN sys.database_principals DP2 ON ...
WHERE DP1.type = 'R' -- R = database role
ORDER BY DP1.name
Masking Functions Reference
partial(p,"pad",s)
Expose p chars prefix, pad in middle, s chars suffix
partial(1,".....",4)
→ 1.....7890
email()
Shows first letter + XXX@XXXX.com format
→ sXXX@XXXX.com
default()
0 for numeric, '' for strings, 1900-01-01 for dates
→ 0 or xxxx
random(lo,hi)
Random number in range (numeric only)
random(1,100)
→ 42
Warning: UNMASK privilege allows the user to also modify masked columns. Dynamic Data Masking hides data from SELECT — it does NOT prevent writes. Use DENY INSERT, UPDATE separately if write-protection is needed.

Backup Encryption (reference) 
USE master
CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'h6twqPNO'
GO
OPEN MASTER KEY DECRYPTION BY PASSWORD = 'h6twqPNO'
BACKUP MASTER KEY TO FILE = 'C:\install\exportedmasterkey'
    ENCRYPTION BY PASSWORD = 'h6twqPNOh6twqPNO'
GO
CREATE CERTIFICATE backup_cert_master
    WITH SUBJECT = 'NW DB backup', EXPIRY_DATE = '20301031'
BACKUP CERTIFICATE backup_cert_master TO FILE = 'C:\install\exportedcert'
GO
BACKUP DATABASE northwind TO DISK = 'C:\install\nw_enc.bak'
WITH COMPRESSION,
     ENCRYPTION (
         ALGORITHM = AES_256,
         SERVER CERTIFICATE = backup_cert_master
     ),
     STATS = 10
GO
-- ⚠ The backup is unreadable on another instance
-- unless backup_cert_master is also restored there.

SSL Configuration (reference)
  1. Create self-signed cert: certreq -new -f certparam.inf local.cer
  2. Grant MSSQL$INSTANCE user full control of the cert private key via mmc
  3. SQL Server Configuration Manager → Protocols → Force Encryption = Yes → restart
  4. Verify: SELECT encrypt_option, auth_scheme FROM sys.dm_exec_connections
-- Confirm encryption:
-- encrypt_option = TRUE  → SSL active
-- auth_scheme    = SQL   → SQL Server authentication
Always Encrypted
Encryption/decryption happens in the client. Keys are stored in the Windows certificate store or Azure Key Vault — outside SQL Server.
Randomized
Different ciphertext each time. No GROUP BY / JOIN / filter on encrypted column
Deterministic
Same plaintext → same ciphertext. Allows equality filters. ECB-attack risk
Even a sysadmin sees only ciphertext in SELECT results. Decryption requires the column master key from the client store.
