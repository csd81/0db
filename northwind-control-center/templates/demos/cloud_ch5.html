Cloud Book — Chapters 5 & 6

Google Cloud Platform (GCP)

Compute Engine · Cloud Storage · App Engine · Cloud Shell · gcloud CLI · Java Client Library. 200+ countries · 24 regions · 73 zones · 100+ products.

GCP VM Instance Groups

Compute Engine VMs are grouped by intended workload. vCPUs: 1–96. RAM: up to 1.4 TB per VM. Charged per-second at a flat rate determined by core count and RAM size.

E2 / N1 / N2 / N2D — General Purpose
Good price-to-performance balance. Web servers, database applications.
N1 = Intel, N2 = 2nd-gen Intel, N2D = AMD EPYC, E2 = shared-core budget
C2 — Compute Optimised
High-performance computing, gaming applications, engineering design (CAD/CAE), HPC.
M2 — Memory Optimised
Large memory footprints. In-memory databases, real-time data analytics, SAP HANA.
A2 — Accelerator (NVIDIA GPU)
AI training/inference, massively parallel HPC. Multiple NVIDIA A100 GPUs per VM.

Advanced Compute Engine Features
Security & Isolation
Sole-Tenant Nodes
A physical server node runs only your Compute Engine instances — no other tenants. Equivalent to having a dedicated server in the cloud. Required for strict compliance scenarios (HIPAA, FedRAMP).
Confidential VMs
Traditional cloud: encrypted at rest + in transit, but decrypted in memory during execution. Confidential Computing performs calculations inside the CPU without ever decrypting data — memory stays encrypted even while the CPU operates on it (AMD SEV technology).
Availability & Reliability
Live Migration
Servers must occasionally stop for maintenance (software updates, hardware replacement). GCP's Live Migration transparently moves your running VM to another physical host in the same zone without rebooting or user intervention — true 24/7 operation guarantee.
GCP Resource Hierarchy — Global · Regional · Zonal

24 regions · 73 zones. Zone = single failure domain. <1 ms intra-zone round-trip latency.

Global
Disk images, snapshots, networks — available everywhere
Regional
Static external IPs — scoped to us-central1, europe-west1, asia-east1…
Zonal
VM instances, persistent disks — most resources live here
Replicating applications across zones in the same region provides redundancy without inter-region latency. Multi-region deployment improves global performance and disaster recovery.
Cloud Storage — Storage Classes

Object storage (like S3). Unlimited capacity. 11-nines durability (99.999999999%). Geo-redundancy available: multi-region or dual-region. Accessed globally at low latency.

Standard Storage No minimum
Frequently accessed data — websites, streaming video, mobile apps. Best performance and availability.
Nearline Storage ≥ 30 days
Low-cost for infrequent access — backup, long-tail multimedia content. Retrieval fee applies.
Coldline Storage ≥ 90 days
Very low cost for disaster recovery data. 90-day minimum storage commitment.
Archive Storage ≥ 365 days
Lowest cost. Designed for regulatory data archival — 365-day minimum between read operations.

GCP Network Services

GCP's private global fibre backbone connects all data centres — traffic between regions travels over Google's own network, not the public internet, providing better throughput and lower latency.

Cloud Interconnect
Connect any external infrastructure to GCP. Dedicated (10/100 Gbps) or Partner Interconnect. Lower latency than internet VPN, more predictable throughput.
Cloud VPN
Encrypted IPsec tunnel between on-premises networks and GCP VPC. HA VPN provides 99.99% SLA with two tunnels in active/active configuration.
Virtual Private Cloud
Logically isolated network within GCP. Can be regional or global (GCP VPCs span regions natively, unlike AWS where VPCs are region-scoped). Subnets are regional.
Cloud Load Balancing
Global, software-defined load balancing with no warm-up time. Automatically distributes traffic across Compute Engine, GKE, App Engine. HTTP(S), TCP, SSL proxy, UDP types.
Cloud Armor
DDoS protection and web application firewall (WAF). OWASP Top 10 protection, IP-based allow/deny, adaptive protection (ML-based DDoS mitigation). Integrated with Cloud Load Balancing.

gcloud CLI — Command Hierarchy

The gcloud CLI is part of the Google Cloud SDK, installed locally or available in Cloud Shell. Commands follow a hierarchical structure: gcloud <service> <resource> <verb> [--flags]

Command structure:
gcloud <group> <sub-group> <command> [--flags]
gcloud compute instances list --filter=...
gcloud projects list --format=...
gcloud storage buckets create gs://...
gcloud run deploy --image=...
bash — List Compute Engine instances by zone
gcloud compute instances list \
    --filter="zone:us-central1-a"

# Output: NAME  ZONE           MACHINE_TYPE  STATUS
# web-1  us-central1-a  e2-medium     RUNNING
# db-1   us-central1-a  n2-standard-4 RUNNING
bash — List projects with formatted table output
gcloud projects list \
    --format="table(name, project_id, \
               createTime.date(tz=LOCAL))"

# NAME           PROJECT_ID        CREATE_TIME
# My App         my-app-123456     2024-01-15 09:30:00
# Dev Sandbox    dev-sandbox-789   2023-11-02 14:20:00
bash — Other useful gcloud patterns
# Start / stop a VM
gcloud compute instances start INSTANCE_NAME \
    --zone=us-central1-a
gcloud compute instances stop INSTANCE_NAME \
    --zone=us-central1-a

# SSH into a VM (handles key management automatically)
gcloud compute ssh INSTANCE_NAME --zone=us-central1-a

gcloud compute images list --filter="family:debian"

# Set default region/zone (avoids repeating --zone)

Compute Engine — Java Client Library

GCP Java Client Library pattern: instantiate a service client via XxxOptions.getDefaultInstance().getService(), then call methods on it. Long-running operations return an Operation object — call .waitFor() to block until complete.

Java — List all Compute Engine instances
// 1. Instantiate the Compute client
Compute compute =
    ComputeOptions.getDefaultInstance().getService();

// 2. List instances (paginated)
Page<Instance> instancePage = compute.listInstances();
Iterator<Instance> it = instancePage.iterateAll();
while (it.hasNext()) {
    Instance i = it.next();
    System.out.printf("Instance: %s%n",
        i.getInstanceId().getInstance());
    System.out.printf("Details: %s%n%n", i);
}
Java — Start a Compute Engine instance
Compute compute =
    ComputeOptions.getDefaultInstance().getService();

// Returns immediately — async operation
Operation operation = compute.start(instanceId);
if (operation == null) {
    System.out.printf("Error: Instance %s not found%n",
                      instanceId);
    return;
}
// Block until the start operation completes
try {
    operation = operation.waitFor();
} catch (InterruptedException | TimeoutException e) {
    e.printStackTrace();
}
Java — Stop a Compute Engine instance
Compute compute =
    ComputeOptions.getDefaultInstance().getService();

Operation operation = compute.stop(instanceId);
if (operation == null) {
    System.out.printf("Error: Instance %s not found%n",
                      instanceId);
    return;
}
try {
    operation = operation.waitFor();
} catch (InterruptedException | TimeoutException e) {
    e.printStackTrace();
}

Cloud Storage — Java Client Library

Cloud Storage client uses StorageOptions.newBuilder().setProjectId(projectId).build().getService(). Buckets are created with a BucketInfo builder specifying storage class and location.

Java — Create a Cloud Storage bucket
import com.google.cloud.storage.*;

public class CreateBucket {
  public static void create(
      String projectId, String bucketName) {

    // 1. Build the Storage client
    Storage storage = StorageOptions.newBuilder()
        .setProjectId(projectId)
        .build()
        .getService();

    // 2. Choose storage class and location
    StorageClass storageClass = StorageClass.COLDLINE;
    String location = "ASIA";   // multi-region

    // 3. Create the bucket
    Bucket bucket = storage.create(
        BucketInfo.newBuilder(bucketName)
            .setStorageClass(storageClass)
            .setLocation(location)
            .build());

    System.out.printf("Created bucket %s in %s (%s)%n",
        bucket.getName(),
        bucket.getLocation(),
        bucket.getStorageClass());
  }
}
Java — List all Cloud Storage buckets
import com.google.api.gax.paging.Page;
import com.google.cloud.storage.*;

public class ListBuckets {
  public static void list(String projectId) {
    Storage storage = StorageOptions.newBuilder()
        .setProjectId(projectId)
        .build()
        .getService();

    // Page<Bucket> handles pagination automatically
    Page<Bucket> buckets = storage.list();
    for (Bucket bucket : buckets.iterateAll()) {
        System.out.println(bucket.getName());
    }
  }
}
StorageClass Options
StorageClass.STANDARD Frequently accessed data
StorageClass.NEARLINE Accessed < once per 30 days
StorageClass.COLDLINE Accessed < once per 90 days
StorageClass.ARCHIVE Accessed < once per 365 days
Location strings: "US" (multi-region), "EUROPE", "ASIA", or a specific region like "us-central1".
