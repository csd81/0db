The GenAI Cloud & the Vectorization of Everything

ChatGPT's launch in November 2022 made LLMs a mainstream infrastructure concern overnight. Enterprises discovered that stateless LLMs couldn't access their proprietary data — and that SQL keyword search couldn't bridge the gap. A new database category emerged to solve it.

⚠ The Breaking Point: LLMs Are Stateless Amnesiacs

An LLM has no memory. Each API call is a blank slate. To make ChatGPT answer questions about your company's 500,000 internal documents, you'd need to paste all 500,000 documents into every prompt — at $20 per million tokens, this is economically impossible. Even if cost were free, it physically doesn't fit in the context window.

SQL Keyword Search Fails

SQL LIKE '%machine learning%' can't find a document that uses the phrase "neural network training" instead. It has zero semantic understanding. Relevant content is missed; irrelevant exact-matches are returned.

Elasticsearch Keyword Search Also Fails

Elasticsearch's BM25 ranking is better than SQL LIKE, but it's still bag-of-words. "What is the capital of France?" and "Paris is the capital city of the French Republic" have low BM25 similarity despite being a perfect question-answer pair.

✦ Vector Databases: Geometry-Based Semantic Search

A text embedding model (like OpenAI's text-embedding-3-large) maps any piece of text to a point in 1,536-dimensional space. Semantically similar texts land geometrically close to each other. A vector database stores these points and can find the N closest neighbors to a query vector in milliseconds — using Approximate Nearest Neighbor (ANN) algorithms like HNSW or IVF.

# RAG: Retrieval-Augmented Generation pipeline
# Solve LLM statelessness by injecting retrieved context into the prompt

import openai, pinecone

## OFFLINE: Index your documents (run once)
def index_documents(docs: list[str]):
    for chunk in chunk_text(docs, size=512):  # 512-token chunks
        embedding = openai.embeddings.create(
            model="text-embedding-3-large",
            input=chunk
        ).data[0].embedding                    # → 1,536-dim float32 vector
        pinecone_index.upsert([(chunk_id, embedding, {"text": chunk})])

## ONLINE: Answer a user query (every request)
def rag_query(user_question: str) -> str:
    # 1. Embed the question in the same vector space
    q_vec = openai.embeddings.create(
        model="text-embedding-3-large", input=user_question
    ).data[0].embedding

    # 2. ANN search: find top-5 geometrically closest document chunks
    #    Distance metric: cosine similarity in 1,536-dim space
    matches = pinecone_index.query(vector=q_vec, top_k=5, include_metadata=True)
    context = "\n\n".join([m.metadata["text"] for m in matches.matches])

    # 3. Inject retrieved context into the LLM prompt
    response = openai.chat.completions.create(
        model="gpt-4o",
        messages=[
            {"role": "system",  "content": f"Answer using this context:\n\n{context}"},
            {"role": "user",    "content": user_question}
        ]
    )
    return response.choices[0].message.content
    # LLM now "knows" the relevant documents without seeing all 500K of them

✦ GPU Cloud Orchestration: H100 Clusters as a Service

Training GPT-4 required ~25,000 Nvidia A100s for ~90–100 days. The fundamental unit of cloud compute shifted from CPU cores to GPU interconnect bandwidth. Standard cloud networks (10 Gbps Ethernet) are too slow — GPU-to-GPU communication requires 400 Gbps InfiniBand to avoid the interconnect becoming the bottleneck when 80GB HBM3 memory must be synchronized across thousands of GPUs every forward/backward pass.

Standard Cloud GPU (AWS p4d.24xlarge)
  • 8× A100 GPUs (80GB HBM2e each)
  • 400 Gbps Ethernet between instances
  • NVLink within instance: 600 GB/s
  • Good for: fine-tuning, inference, small training
  • ~$32/hr on-demand
CoreWeave H100 Cluster (foundation model)
  • 10,000+ H100 SXM5 (80GB HBM3)
  • 3.2 Tbps InfiniBand HDR between nodes
  • NVLink 4.0: 900 GB/s within node
  • Good for: pre-training, RL post-training
  • ~$2.50/H100/hr × 10K = $25K/hr at scale

The Verdict

Era 5 is the first time a new database category (vector databases) emerged primarily to serve a single new workload (RAG for LLMs). The speed of adoption — Pinecone went from founding to $750M valuation in 18 months — reflects how suddenly the semantic search gap became a blocking problem for an entire industry. The deeper shift is architectural: AI is now a first-class infrastructure concern, not a research appendage. Every database vendor (PostgreSQL, Redis, MongoDB, Elasticsearch) has added vector search. The question for the next era is not whether AI will be in the stack — it's whether it will run in the cloud or on the device.
