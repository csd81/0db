Oracle Database

Relational OLTP · SQL + PL/SQL · MVCC (SCN) · #1 enterprise RDBMS by market share

Founded by Larry Ellison in 1977, Oracle Database has been the dominant commercial RDBMS for nearly five decades. It consistently ranks #1 on DB-Engines and powers the majority of the Fortune 500 — SAP ERP systems, global banking core ledgers, telco billing, and government infrastructure. Oracle's strength lies in its unmatched feature set: Real Application Clusters (RAC) for shared-disk HA, Data Guard for disaster recovery, Exadata engineered systems for extreme throughput, and decades of optimizer maturity. The flip side is complexity and licensing cost, which has driven the industry's "Oracle to PostgreSQL" migration wave.

Core Abstractions

Tablespace — logical storage container mapping to physical datafiles. Every object lives in a tablespace (SYSTEM, SYSAUX, USERS, TEMP, UNDO).

Schema — namespace owned by a database user. Oracle conflates user and schema (CREATE USER = CREATE SCHEMA).

Segment / Extent / Block — physical hierarchy: blocks (8KB) → extents → segments (one per table/index).

ROWID — unique physical address for every row (file#.block#.row# encoded in base-64). Fastest access method.

Partitioning — RANGE, HASH, LIST, and COMPOSITE (sub)partitioning. Partition pruning eliminates unneeded I/O automatically.

Materialized Views — pre-computed query results stored as real tables, refreshed ON COMMIT or ON DEMAND. Query Rewrite rewrites queries automatically.

Schema Example 
-- Partitioned orders table
CREATE TABLE orders (
    order_id   NUMBER        GENERATED ALWAYS AS IDENTITY,
    cust_id    NUMBER        NOT NULL,
    status     VARCHAR2(20)  DEFAULT 'PENDING',
    total      NUMBER(12,2),
    created_at DATE          DEFAULT SYSDATE,
    CONSTRAINT pk_orders PRIMARY KEY (order_id)
)
PARTITION BY RANGE (created_at)
INTERVAL (NUMTOYMINTERVAL(1,'MONTH'))
(PARTITION p0 VALUES LESS THAN
    (DATE '2024-01-01'));

-- JSON column (21c+)
ALTER TABLE orders ADD payload CLOB
    CONSTRAINT chk_json CHECK (payload IS JSON);

-- Global index survives partition ops
CREATE INDEX idx_orders_cust
    ON orders(cust_id) GLOBAL;
SQL + PL/SQL Primer 
-- 1. Hierarchical query (CONNECT BY) — org chart
SELECT LPAD(' ', 2*(LEVEL-1)) || last_name AS org_chart,
       employee_id, manager_id, LEVEL AS depth
FROM employees
START WITH manager_id IS NULL
CONNECT BY PRIOR employee_id = manager_id
ORDER SIBLINGS BY last_name;

-- 2. Analytic window function
SELECT customer_id, order_date, total,
       SUM(total) OVER (
           PARTITION BY customer_id
           ORDER BY order_date
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS running_total
FROM orders;

-- 3. MERGE (upsert)
MERGE INTO inventory tgt
USING staging src ON (tgt.product_id = src.product_id)
WHEN MATCHED THEN UPDATE SET
    tgt.qty = src.qty, tgt.updated = SYSDATE
WHEN NOT MATCHED THEN INSERT
    (product_id, qty, updated)
    VALUES (src.product_id, src.qty, SYSDATE);

-- 4. Flashback query — time travel
SELECT * FROM orders
AS OF TIMESTAMP (SYSTIMESTAMP - INTERVAL '1' HOUR)
WHERE order_id = 42;

-- 5. PL/SQL stored procedure
CREATE OR REPLACE PROCEDURE transfer_funds(
    p_from  IN NUMBER,
    p_to    IN NUMBER,
    p_amt   IN NUMBER
) AS
BEGIN
    UPDATE accounts SET balance = balance - p_amt
    WHERE account_id = p_from;
    UPDATE accounts SET balance = balance + p_amt
    WHERE account_id = p_to;
    COMMIT;
EXCEPTION
    WHEN OTHERS THEN
        ROLLBACK;
        RAISE;
END;

Killer Use Cases

ERP Systems (SAP on Oracle) — SAP's R/3 and S/4HANA are certified on Oracle. The combination powers payroll, procurement, and manufacturing for thousands of enterprises.

Banking Core Ledgers — ACID + serializable isolation + audit trails make Oracle the go-to for general ledgers, loan origination, and payment processing in global banks.

Telco Billing — call data records at billion-row scale, partitioned by month, with materialized views for billing aggregates. Oracle's optimizer handles this well.

Government & Healthcare — regulatory compliance, HIPAA audit logs, Label-Based Access Control (LBAC) for row-level security in classified systems.

Performance

OLTP latency: <1ms (SSD, buffer cache hit)

Throughput: Millions of TPS on Exadata

Exadata Smart Scan: Column filter offload to storage cells

In-Memory Column Store: 100x analytics speedup on same data

RAC: Active-active nodes, shared ASM storage

Memory (SGA)

Buffer Cache — caches data blocks in memory using LRU. db_cache_size is the most impactful tuning knob.

Shared Pool — parsed SQL, execution plans (library cache), and data dictionary cache. Hard parses are expensive; use bind variables.

Redo Log Buffer — circular buffer holding redo entries before LGWR flushes them to disk. Every DML writes here first.

Large Pool / Java Pool — for parallel query, RMAN, Java stored procedures.

Background Processes

LGWR — Log Writer: flushes redo buffer to online redo logs on COMMIT.

DBWn — Database Writers: write dirty blocks from buffer cache to datafiles.

CKPT — Checkpoint: updates control file and datafile headers with SCN.

SMON — System Monitor: crash recovery, temp segment cleanup.

ARCn — Archiver: copies filled redo logs to archive log destination (needed for Data Guard).

HA & DR

Real Application Clusters (RAC) — multiple instances share one database on Oracle ASM storage. Active-active: all nodes can read/write.

Data Guard — physical or logical standby databases. Synchronous (Maximum Protection) or async (Maximum Performance). Fast failover via broker.

GoldenGate — log-based replication for heterogeneous targets (Oracle→PostgreSQL, Oracle→Kafka). Used for zero-downtime migrations.
