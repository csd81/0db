Blockchain — Proof of Work Demo

Idle 0 blocks 0 forks

Bitcoin's Proof-of-Work consensus requires that a block's SHA-256 hash start with a required number of zero bits, forcing miners to try billions of nonces until one satisfies the constraint. Each additional leading zero roughly doubles the expected computation, which is how the network maintains ~10-minute block times by raising difficulty as total hash power grows. Three peer nodes race to mine each block; when two finish simultaneously, a fork occurs and the first-broadcast block wins while the other becomes an orphan. The chain of linked hashes makes tampering with any historical block immediately detectable because every subsequent block's hash would change.

How it works: Three peers race to mine each block by finding a nonce where SHA-256(prev_hash | txs | peer | nonce) starts with 00. The winner broadcasts to the rqlite network ledger. If two peers finish during the propagation window → fork — the first-seen block wins, the other is orphaned. Unlike traditional DB replication, no central authority decides the winner — cryptographic proof does.
