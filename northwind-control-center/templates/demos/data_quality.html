Technique — GROUP BY double-count
To test whether column B is functionally dependent on column A (i.e. knowing A always tells you B), run two queries: group by A alone → count groups (N), then group by A + B together → count groups (M). If N = M the pairing never splits — B is likely functionally dependent on A. A hidden FD can indicate an unmodelled business rule and a potential 3NF violation (transitive dependency).

SQL Server Data Quality Services (DQS) — Architecture & Workflow




  

    Three DQS Databases

    DQS_MAIN — the engine: stored procedures, Knowledge Bases, reference value sets.
    Roles: dqs_administrator, dqs_kb_editor, dqs_kb_operator.

    DQS_PROJECTS — state of all active cleansing and matching projects.

    DQS_STAGING_DATA — temporary storage for data being cleaned and its results.
    Custom views like vi_places are created here to feed Knowledge Discovery.
  

  

    Knowledge Base Structure

    A KB contains one or more Domains.
    Each domain is defined by:

    • Name & data type (e.g. string)

    • Normalisation rules (uppercase, trim)

    • Domain values — leading value + synonyms (e.g. Mühlheim ← Muehlheim)

    • Domain rules — like CHECK constraints (e.g. BirthDate > 1920-01-01)

    • Term-based relations — substring replacements (e.g. Ltd. → Limited Company)

    Composite domains link semantically related fields (City + State + Country).
  

  

    Cleansing Result States

    After a cleansing project runs, each value lands in one of five states:

    ✓ Correct — found exactly in domain values

    ↻ Corrected — auto-fixed (similarity > Min Score for Auto Correction)

    ? Suggested — similar match found, awaits approval

    + New — conforms to domain rules but not in value set

    ✗ Invalid — violates a domain rule
  

  

    Two Project Types

    Basic cleansing — corrects individual field values using a KB domain.
    Min Score for Suggestions must be < Min Score for Auto Correction.


    Identity mapping / de-duplication — identifies duplicate records using
    configurable distance measures. Not covered in this course.


    Pre-installed KBs: Country/Region, US-Counties, US-Last Names.
    Demo: cleaned 18,484 LastName values in DimCustomer — 100% completeness, 98% accuracy.

View definition (run on DQS_STAGING_DATA):

      
USE DQS_STAGING_DATA;
GO

CREATE VIEW vi_places AS
SELECT DISTINCT
    City,
    StateProvinceName  AS StateProvince,
    EnglishCountryRegionName AS CountryRegion
FROM AdventureWorksDW2019.dbo.DimGeography;
GO

    Data Profiling (automated)

    Profiling goes beyond manual GROUP BY queries: it automatically computes null ratios, value distributions,
    candidate keys, regex patterns, and potential foreign keys across all columns of a table.
    In SQL Server, the SSIS Data Profiling Task saves results as XML (reviewed in the Data Profile Viewer).
    The interactive Column Profiler above implements a lightweight subset of this for live exploration.
  

  

    MDM vs DQ

    Data Quality (DQ) improves existing data (correct, deduplicate, standardise).

    Master Data Management (MDM) establishes a single authoritative source for key business entities
    (customers, products, employees) across all systems — requiring a Data Governance framework and
    Data Stewards responsible for specific data domains. MDM solutions are not covered in this course.
