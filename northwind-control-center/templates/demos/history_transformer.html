Three Fatal Flaws of RNNs

1. Sequential = Unparallelizable
RNNs process tokens one at a time: hidden state h₁ feeds into h₂, which feeds into h₃. To compute h₅, you must have already computed h₄. This creates a strict data dependency chain — you cannot distribute computation across thousands of GPUs because each step depends on the previous step's output. Training a large RNN on the internet takes years, not weeks.
2. Vanishing Gradients = Forgetting
During backpropagation, gradients are multiplied at each step. With sequences of 500+ tokens, gradients shrink exponentially (or explode). The model literally forgets the beginning of a long sentence by the time it processes the end. LSTMs (a fix) helped but didn't solve it — still O(sequence_length) data dependencies.
3. Fixed Context Window
RNN hidden state has fixed dimensions — it must compress the entire history of a paragraph into a single vector of, say, 512 floats. Information is lost. Context windows of more than ~200 tokens became unreliable. This made it impossible to reason over books, codebases, or long documents.
Self-Attention: Seeing Every Word Simultaneously
The Core Idea
For each word in a sentence, compute three vectors: a Query (what am I looking for?), a Key (what do I represent?), and a Value (what information do I carry?). Then: each word's Query is compared (dot product) against every other word's Key to produce an attention score — how relevant is that word to me? Softmax normalizes scores into weights. The output is a weighted sum of all Values.
The Bank Example
Sentence: "The bank is on the river bank"
First "bank": high attention to "river" → financial meaning suppressed
Second "bank": high attention to "river" → riverbank confirmed
Both resolved in ONE forward pass, not sequentially
Why This Enables Parallelism
All tokens' Q, K, V matrices are computed simultaneously (matrix multiplication, highly parallelizable on GPU CUDA cores). Attention scores for all word pairs computed in one batched matrix operation. No data dependency between tokens — GPU computes all 512 tokens' representations in parallel. A 10,000-GPU cluster can now train on the entire internet in weeks, not decades.
Self-Attention: The Math (simplified)
import numpy as np

# For sentence: ["The", "bank", "river", "flows"]
# Each token gets 3 learned vectors (d_model=4 for demo)
# In practice: d_model=512 to 8192 depending on model size

Q = np.array([  # Query: what am I looking for?
    [0.1, 0.8, 0.2, 0.1],  # "The"
    [0.9, 0.2, 0.1, 0.8],  # "bank" — asking about context
    [0.1, 0.1, 0.9, 0.3],  # "river" — asking about geography
    [0.2, 0.1, 0.7, 0.1],  # "flows"
])

K = np.array([  # Key: what do I represent?
    [0.1, 0.2, 0.1, 0.1],  # "The"
    [0.8, 0.3, 0.2, 0.7],  # "bank" — represents ambiguous entity
    [0.2, 0.1, 0.9, 0.4],  # "river" — represents water/geography
    [0.1, 0.1, 0.8, 0.2],  # "flows"
])

V = np.array([  # Value: what information do I carry?
    [0.1, 0.1, 0.1, 0.1],
    [0.7, 0.2, 0.1, 0.6],  # "bank" value
    [0.1, 0.1, 0.9, 0.5],  # "river" value (geography signal)
    [0.1, 0.1, 0.8, 0.3],
])

d_k = Q.shape[-1]

# Step 1: Attention scores — every token vs every token
# Q × K^T: matrix multiply (fully parallelizable on GPU)
scores = Q @ K.T / np.sqrt(d_k)

# Step 2: Softmax → probability distribution over tokens
def softmax(x):
    e = np.exp(x - x.max(axis=-1, keepdims=True))
    return e / e.sum(axis=-1, keepdims=True)

attn_weights = softmax(scores)
# "bank" row: high weight on "river" → geography context wins

# Step 3: Weighted sum of Values
output = attn_weights @ V
# "bank" output now contains "river"'s value information
# → model understands this is a riverbank, not financial

# The KEY insight: ALL of this is one matrix operation.
# 512 tokens → 512×512 attention matrix → computed in parallel
# vs RNN: 512 sequential hidden state updates (unparallelizable)
From Architecture to Industry: The Transformer Family Tree
Multi-Head Attention
Run 8–64 attention heads in parallel, each learning different relationship types (syntax, semantics, coreference). Concatenate outputs. One head might learn "subject-verb" relationships; another "adjective-noun." Richer representations than a single attention pass.
Positional Encoding
Self-attention has no inherent notion of word order (it just sees a set of tokens). Inject position information by adding a sinusoidal position vector to each token embedding. The model learns to distinguish "dog bites man" from "man bites dog."
Feed-Forward + Residual Connections
Each attention layer followed by a position-wise feed-forward network (two linear layers + ReLU). Residual connections (x + sublayer(x)) prevent vanishing gradients — gradients have a "highway" to flow through. Layer Normalization stabilizes training.
Encoder vs Decoder
Original paper: Encoder (BERT-style) reads full input bidirectionally. Decoder (GPT-style) generates text left-to-right with masked self-attention (can't see future tokens). Encoder-Decoder (T5, original Transformer) for translation. GPT uses decoder-only → simplifies scaling to 1T+ parameters.
Transformer evolution: same architecture, more parameters + data
BERT (2018)
Google. 340M params. Encoder-only. Bidirectional context. Dominates NLP benchmarks. Still deployed in Google Search.
GPT-2 (2019)
OpenAI. 1.5B params. Decoder-only. OpenAI initially refused release "too dangerous." Generated coherent paragraphs — first mass-market AI writing.
GPT-3 (2020)
175B params. In-context learning emerges — give examples in the prompt, model generalizes. ChatGPT ancestor. Few-shot learning.
ChatGPT (2022)
GPT-3.5 + RLHF (human feedback). 100M users in 60 days. Mass market AI. Instruction-following emerges as trainable behavior.
GPT-4o+ / Claude 4
Multimodal: text + image + audio. Long context windows (200K+ tokens). Mixture of Experts (MoE). Same Transformer core, massively scaled.
AlphaFold2
DeepMind. Same Transformer on protein sequences instead of text. Solved 50-year-old protein folding problem in 2021. 200M proteins predicted.
🔬 The Scaling Law: Why Bigger Is Always Better (Within a Law)
Chinchilla Scaling Law (2022)
DeepMind paper: optimal performance requires equal scaling of parameters AND training data. Rule: N parameters → train on 20×N tokens. GPT-3 (175B params, 300B tokens) was under-trained. Chinchilla 70B trained on 1.4T tokens outperformed 280B GPT-3 on most benchmarks. Data matters as much as model size.
Flash Attention (2022)
The O(n²) cost of attention becomes prohibitive at long contexts (100K tokens → 10B attention pairs). Flash Attention: reorders computation to be IO-aware — avoids materializing the full n×n attention matrix in GPU HBM (slow). Instead computes attention in tiles that fit in fast SRAM. 5-10× faster, enables 1M+ token context windows.
Mixture of Experts (MoE)
Instead of all parameters active for every token: divide into "experts" (sub-networks). A router selects 2-8 experts per token. GPT-4 rumored to be 8 experts × 220B params = 1.76T total, but only 440B active per token. Same quality as dense model, 5× cheaper inference. Mistral 8×7B, Gemini Ultra use MoE.

The Lasting Verdict

"Attention Is All You Need" (Vaswani et al., 2017) is arguably the most consequential machine learning paper ever published. Eight of the authors have since left Google to found competing AI companies (OpenAI, Cohere, Adept, Character.AI, Inceptive). The Transformer architecture solved the sequential processing bottleneck, enabling training on internet-scale data with thousands of GPUs. Every major AI system that exists today — ChatGPT, Claude, Gemini, Sora, DALL-E, AlphaFold, Whisper, Stable Diffusion — runs on some variant of the Transformer. It didn't just change AI; it changed science, software, and arguably civilization.

🎓 RLHF: Turning a Model Into an Assistant
A pre-trained Transformer predicts next tokens — it doesn't "follow instructions." RLHF (Reinforcement Learning from Human Feedback): 1) supervised fine-tuning on Q&A pairs, 2) human raters score model outputs, 3) train a "reward model" to predict human preference, 4) use PPO (RL algorithm) to optimize the base model to maximize the reward model's score. This is what makes ChatGPT helpful rather than a text predictor.
💻 The GPU Monopoly
Training GPT-4 estimated cost: $50–100M. Primary bottleneck: NVIDIA H100 GPUs ($30K each). NVIDIA's CUDA ecosystem (software toolkit for GPU computing) creates a 10-year moat — competing GPU hardware exists but lacks CUDA's library ecosystem. NVIDIA's market cap went from $300B (2022) to $3T (2024) riding the Transformer wave.
