The Capacity Planning Problem

Every enterprise faces a conflict between two goals:

Goal 1: Minimise infrastructure size
Keep upfront + operational cost at minimum
Goal 2: Enough resources for peak load
Profitable operation requires serving extreme burst loads
Cloud Solution
Plan for average load. Burst to cloud at peak times. No wasted over-provisioning.

Per-second billing makes burst workloads very economical — a 35-second job costs exactly 35 seconds, not a full hour.
Enterprise preference
Businesses favor fixed-price on-demand: predictable for budgeting and accounting. Spot pricing is for engineers optimizing batch workloads, not for CFOs planning quarters.
Programming Language Support

Two conflicting requirements shape cloud language support:

Users want
Complete platform independence AND keep existing source code working without modification
Providers want
Support as few languages as possible — each supported language = replicate entire API, testing, maintenance, future development
Result: Most Supported Languages
