Azure Synapse Analytics

Unified analytics · Dedicated SQL + Serverless SQL + Spark · One workspace

Azure Synapse Analytics (2020, evolved from Azure SQL Data Warehouse) combines multiple analytics engines in a single workspace: Dedicated SQL Pools (MPP warehouse), Serverless SQL (query files in ADLS Gen2 without loading them), Apache Spark (notebooks for data engineering and ML), and Power BI for visualization. Synapse Link enables zero-ETL analytical reads from Cosmos DB, Azure SQL, and Dataverse — the analytical replica is maintained automatically.

Four Engines, One Workspace
Dedicated SQL Pool

MPP T-SQL warehouse. Provisioned DWU capacity. Clustered Columnstore Indexes. Best for predictable, heavy warehouse queries.

Serverless SQL Pool

Query CSV, Parquet, JSON, Delta in ADLS Gen2 directly. Pay per TB scanned. No loading needed. OPENROWSET().

Apache Spark

Managed Spark clusters. Python/Scala/R/SQL notebooks. Delta Lake support. Synapse ML library. Data engineering pipelines.

Synapse Link

Zero-ETL from Cosmos DB, Azure SQL, Dataverse. Analytical replica auto-maintained. No pipeline = no lag from ETL.

Dedicated SQL Pool — Distribution & CCI 
-- Distribution styles (same concept as Redshift)
CREATE TABLE fact_sales (
    sale_id      BIGINT,
    product_id   INT,
    customer_id  INT,
    sale_date    DATE,
    quantity     INT,
    revenue      DECIMAL(10,2)
)
WITH (
    DISTRIBUTION = HASH (customer_id),   -- co-locate by customer for JOINs
    CLUSTERED COLUMNSTORE INDEX          -- CCI: columnar storage, best compression
);

-- Dimension table: replicate to all nodes (small table)
CREATE TABLE dim_products (...)
WITH (DISTRIBUTION = REPLICATE, HEAP);

-- Staging table: no distribution needed
CREATE TABLE stg_raw_events (...)
WITH (DISTRIBUTION = ROUND_ROBIN, HEAP);

-- PolyBase: load from ADLS Gen2
CREATE EXTERNAL DATA SOURCE adls_source
WITH (TYPE = HADOOP, LOCATION = 'abfss://container@account.dfs.core.windows.net');

CREATE EXTERNAL FILE FORMAT parquet_format
WITH (FORMAT_TYPE = PARQUET, DATA_COMPRESSION = 'snappy');

CREATE EXTERNAL TABLE ext_sales
WITH (LOCATION='/sales/', DATA_SOURCE=adls_source, FILE_FORMAT=parquet_format)
AS SELECT * FROM dbo.fact_sales;
Serverless SQL — Query Data Lake 
-- ── OPENROWSET: query files without loading
SELECT TOP 100 *
FROM OPENROWSET(
    BULK 'https://account.dfs.core.windows.net/container/sales/2024/**',
    FORMAT = 'PARQUET'
) AS sales;

-- ── Query CSV with schema
SELECT *
FROM OPENROWSET(
    BULK 'https://account.dfs.core.windows.net/container/events/*.csv',
    FORMAT = 'CSV',
    FIELDTERMINATOR = ',',
    ROWTERMINATOR = '\n',
    FIRSTROW = 2
) WITH (
    event_id   BIGINT  1,
    event_type VARCHAR(50) 2,
    occurred_at DATETIME2  3
) AS events;

-- ── Create External Table (logical view over data lake)
CREATE EXTERNAL TABLE sales_parquet
WITH (
    LOCATION = 'sales/2024/',
    DATA_SOURCE = adls_ds,
    FILE_FORMAT = parquet_fmt
)
AS
SELECT * FROM OPENROWSET(BULK ...) AS r;

-- ── CETAS: Create External Table As Select
-- Transform and save results back to data lake
CREATE EXTERNAL TABLE aggregated_sales
WITH (LOCATION='aggregated/', DATA_SOURCE=adls_ds, FILE_FORMAT=parquet_fmt)
AS
SELECT DATE_TRUNC('month', sale_date) AS month,
       product_id, SUM(revenue) AS total_revenue
FROM sales_parquet
GROUP BY 1, 2;
Synapse Spark + T-SQL Cross-Engine Queries 
# ── Spark notebook (Python) — data engineering pipeline
from pyspark.sql import SparkSession
spark = SparkSession.builder.getOrCreate()

# Read from ADLS Gen2 (Delta Lake)
df = spark.read.format("delta").load("abfss://container@account.dfs.core.windows.net/silver/sales")

# Transform
from pyspark.sql.functions import col, sum, to_date, date_trunc
agg = (df.filter(col("status") == "completed")
         .withColumn("month", date_trunc("month", col("sale_date")))
         .groupBy("month", "product_id")
         .agg(sum("revenue").alias("monthly_revenue")))

# Write to Synapse Dedicated SQL Pool via connector
agg.write.format("com.microsoft.sqlserver.jdbc.spark") \
    .option("url", "jdbc:sqlserver://workspace.sql.azuresynapse.net") \
    .option("dbtable", "gold.monthly_sales") \
    .option("authentication", "ActiveDirectoryIntegrated") \
    .mode("overwrite").save()

-- ── T-SQL: query the Spark-written table
SELECT month, product_id, monthly_revenue,
       monthly_revenue / SUM(monthly_revenue) OVER (PARTITION BY month) AS share
FROM gold.monthly_sales
ORDER BY month DESC, monthly_revenue DESC;

-- ── Synapse Link: query Cosmos DB operational data analytically
-- (columnar replica maintained automatically, zero ETL)
SELECT c.customerId,
       COUNT(*) AS order_count,
       SUM(c.total) AS lifetime_value
FROM OPENROWSET(
    'CosmosDB',
    N'account=myaccount;database=northwind;collection=orders',
    orders
) WITH (customerId VARCHAR(100), total FLOAT, status VARCHAR(20)) AS c
WHERE c.status = 'shipped'
GROUP BY c.customerId
HAVING SUM(c.total) > 1000;
Ideal Use Cases
Enterprise Data Lakehouse

Combine raw data in ADLS Gen2 (queried via Serverless SQL or Spark) with curated data in Dedicated SQL Pool. Data Factory pipelines orchestrate ETL. Unilever global supply chain analytics.

Real-Time Operational Analytics

Synapse Link from Cosmos DB or Azure SQL → zero-ETL analytical queries on operational data. No replication lag from batch ETL pipelines. Used for real-time retail dashboards.

ML Feature Engineering

Spark notebooks for feature computation at scale, writing to Azure ML Feature Store. Synapse ML (MMLSpark) for distributed model training directly in Spark without exporting data.

Architecture: ADLS Gen2 as Common Storage 
┌───────────────────────────────────────────────────────┐
│  Azure Data Lake Storage Gen2 (ADLS)                  │
│  ├── /bronze/  (raw ingestion from Data Factory)      │
│  ├── /silver/  (cleaned, Delta Lake format)           │
│  └── /gold/    (aggregated, query-ready Parquet)      │
├───────────────────────────────────────────────────────┤
│  Synapse Workspace                                    │
│  ├── Serverless SQL ──────── queries /silver, /gold   │
│  ├── Dedicated SQL Pool ──── loads /gold, CCI         │
│  ├── Spark Pools ─────────── processes /bronze→silver │
│  └── Pipelines ───────────── orchestrates flow        │
├───────────────────────────────────────────────────────┤
│  Connected Services                                   │
│  ├── Synapse Link ← Cosmos DB (zero-ETL)              │
│  ├── Synapse Link ← Azure SQL DB (zero-ETL)           │
│  ├── Azure ML ← feature store integration             │
│  ├── Power BI ← DirectQuery on Dedicated Pool         │
│  └── Purview ← data catalog and lineage               │
└───────────────────────────────────────────────────────┘

DWU scaling: pause/resume Dedicated Pool
  (pay only when running, unlike always-on Snowflake)
Dedicated SQL Pool: Control + Compute Nodes 
┌─────────────────────────────────────────────────────────────────┐
│  Control Node                                                   │
│  ├── Query parsing, optimization, DMS coordination              │
│  └── Distributes query fragments to compute nodes               │
│                │                                                │
│       ┌────────┴─────────────────────────────────┐              │
│  ┌────▼────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐│               │
│  │Compute  │ │Compute  │ │Compute  │ │Compute  ││               │
│  │Node 1   │ │Node 2   │ │Node 3   │ │Node ...  ││              │
│  │(60 dist)│ │(60 dist)│ │(60 dist)│ │(60 dist)││               │
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘│               │
│       │           DMS (Data Movement Service)     │             │
│       └──── shuffles/broadcasts data between nodes              │
├─────────────────────────────────────────────────────────────────┤
│  Storage: ADLS Gen2 (60 shards per table, columnar blocks)      │
│           CCI: Rowgroups of 1M rows, compressed                 │
│           Delta stores: small updates before rowgroup rebuild   │
│           Zone maps: min/max per rowgroup for pruning           │
└─────────────────────────────────────────────────────────────────┘

DWU = Data Warehouse Units:
  DW100c  = 1 compute node, 60 distributions
  DW6000c = 60 compute nodes, 1 distribution per node per node

Pause/Resume: control plane stops VMs, storage persists in ADLS
Scale (DWU change): redistributes data across new node count
