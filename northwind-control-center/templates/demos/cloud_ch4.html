Amazon Web Services (AWS)

Largest cloud platform — 245 countries, 24 regions, multiple Availability Zones per region. EC2 · S3 · EFS · EBS · VPC · ELB · Shield · Elastic Beanstalk · AI SaaS · Java SDK.

EC2 Instance Type Groups

Amazon EC2 provides virtual machines grouped by workload profile. The right group is selected based on the bottleneck: CPU, memory, storage I/O, or GPU throughput.

General Purpose
Balanced CPU/memory. Business-critical apps, small/mid DBs, web tier. Instance classes: A1 (ARM Graviton), M4/M5 (Intel Xeon), M5a (AMD EPYC), T2–T4g (burstable).
T-class burstable: runs at baseline, accumulates credits when idle → spends credits during burst. 0 credits = throttled to baseline.
Compute Optimised
High-performance computing, batch processing, video encoding. High vCPU-to-memory ratio.
Memory Optimised
High-performance DBs, distributed in-memory caches, real-time big data analytics.
Storage Optimised
NoSQL databases, data warehousing, distributed file systems. High sequential I/O.
Accelerated (GPU/FPGA)
ML inference/training, graphic rendering, HPC algorithms, gaming servers.
EC2 Variants
Autoscaling Automatically add/remove capacity to meet demand changes — no manual intervention.
Spot Auction pricing on spare capacity — up to 90% savings for fault-tolerant workloads. May be preempted.
Batch Managed job scheduler — no fixed cluster to provision. Dynamic scaling. Ideal for molecular dynamics, genomics, media rendering.

T-Class Burstable Credit Model
CPU usage < baseline → earn credits (stored, up to cap)
CPU usage > baseline → spend credits (burst performance)

Ideal for: workloads with occasional spikes on a low baseline.

Amazon EFS — File Storage
Managed elastic NFS filesystem. Scales to petabytes on demand without disruption. Two classes: Standard and Infrequent Access (IA). Lifecycle Management moves cold files to EFS IA automatically.
Amazon EBS — Block Storage
High-performance block volumes for EC2. 5 volume types: single-digit-ms latency (DBs) to GB/s throughput (Hadoop). Replicated within AZ. Resize without disruption.
Data Security
Encryption everywhere
Data encrypted in transit (TLS) and at rest. S3 supports server-side (3 key management options) and client-side encryption. Encrypted leaks are practically unusable.
IAM (Identity & Access Management)
All data is private-by-default. Sharing requires explicit IAM permission grants. S3 Access Points enable granular, per-application access policies.
VPC Isolation
Virtual Private Cloud provides further network-level isolation. Resources inside a VPC are not reachable from the internet unless explicitly configured.
Reality of cloud data leaks
Most known cloud data leaks result from human configuration errors, not provider breaches. Cloud vendors employ top security experts operating stricter controls than most on-premises IT.
S3 Storage Classes — Hot to Cold

S3 Intelligent-Tiering automatically moves objects across classes based on access pattern — zero user intervention, significant cost savings.

S3 Standard
General-purpose; frequently accessed data. Highest availability and throughput.
S3 Intelligent-Tiering
AI-driven automatic tier movement based on access patterns. No retrieval fee.
S3 Standard-IA
Infrequent Access — lower storage cost, retrieval fee applies. Milliseconds access.
S3 One Zone-IA
Same as Standard-IA but stored in a single AZ. 20% cheaper; no multi-AZ resilience.
S3 Glacier
Long-term archival. Low cost; retrieval: minutes to hours. Durable (11 nines).
S3 Glacier Deep Archive
Lowest cost; retrieval: 12 hours. For data retained 7–10 years. Regulatory compliance.
S3 Outposts
On-premises S3 running on AWS Outpost hardware. Same S3 APIs, local data residency.

AWS Network Services

Network requirements range from a personal backup device to a multi-national running its entire operation on AWS. Services are grouped by role: build, scale, and secure.

Build — Define & Connect Your Network
Amazon VPC
Logically isolated virtual network. Define IP ranges, subnets (public/private), route tables, internet gateways. Foundation of all AWS networking.
AWS Transit Gateway
Central hub connecting multiple VPCs and on-premises networks. Replaces complex VPC peering mesh.
AWS PrivateLink
Private, dedicated (non-shared) connection between your VPC and AWS services or partner services. Traffic never traverses the public internet.
Amazon Route 53
Managed authoritative DNS service. Routes users to internet applications by translating domain names to IPs. Supports health checks and failover routing.
AWS VPN
Encrypted tunnel between on-premises networks and AWS VPC. Remote workers access cloud resources as if on-premises.
Scale — Handle Traffic Growth
Elastic Load Balancing
Automatically distributes incoming traffic across EC2 instances, containers, IP addresses, and Lambda functions. Application, Network, and Gateway load balancer types.
AWS Global Accelerator
Routes users over the AWS global backbone (not public internet) to the nearest healthy endpoint. Improves latency and availability for global applications.
Secure — Protect Network Traffic
AWS Shield
Managed DDoS protection. Standard (free, automatic) protects EC2, ELB, CloudFront. Advanced adds 24/7 DRT support and attack forensics.
AWS WAF
Web Application Firewall protecting against common exploits (SQL injection, XSS, OWASP Top 10). Deployed in front of CloudFront, ALB, API Gateway.
AWS Firewall Manager
Centrally configure and manage WAF rules, Shield Advanced, and VPC security groups across accounts and applications in an AWS Organisation.

PaaS — AWS Elastic Beanstalk

IaaS gives maximum flexibility but forces developers to manage EC2, scaling, load balancing, and health monitoring manually. PaaS eliminates these operational concerns — the developer sees only the application layer.

Elastic Beanstalk Supported Runtimes
Go Java (Tomcat) .NET Node.js PHP Python Ruby
Java: deploy a .war file → EB provisions Apache Tomcat, EC2 instances, ELB, and Auto Scaling automatically. Java Servlets + JSF supported.
Deployment workflow:
1. Developer uploads app (e.g. myapp.war)
2. EB selects platform version → provisions EC2
3. Auto configures: ELB + Auto Scaling + Health Monitoring
4. App goes live — developer manages via EB console
Developer writes code; AWS handles infrastructure.

AWS CLI — Command-Line Access

The AWS CLI provides access to all services from the shell. Supports fuzzy auto-completion, dynamic inline documentation, AWS SSO, and OS shell command execution. Write scripts to automate common operational tasks.

bash — AWS CLI examples
# List running EC2 instances
$ aws ec2 describe-instances

# Start a specific instance
$ aws ec2 start-instances \
    --instance-ids i-1348636c

# Publish an SNS alert message
$ aws sns publish \
    --topic-arn arn:aws:sns:us-east-1:546419318123:OpsError \
    --message "Script Failure"

# Receive messages from SQS queue
$ aws sqs receive-message \
    --queue-url https://queue.amazonaws.com/546419318123/Test

# Display help for any service
$ aws ec2 help
$ aws s3 help

Java SDK — Create & Monitor EC2 Instances

The Java SDK uses a fluent builder pattern throughout. Every AWS service has a corresponding client class (e.g. Ec2Client). Requests are immutable builder objects; responses are typed.

Java — Create an EC2 instance
String name  = args[0];   // instance name tag
String amiId = args[1];   // OS image ID (e.g. ami-0abcdef1)

// 1. Build the EC2 client for us-west-2
Ec2Client ec2 = Ec2Client.builder()
    .region(Region.US_WEST_2)
    .build();

// 2. Describe the launch configuration
RunInstancesRequest runRequest = RunInstancesRequest.builder()
    .imageId(amiId)
    .instanceType(InstanceType.T1_MICRO)
    .maxCount(1)
    .minCount(1)
    .build();

// 3. Launch the instance
RunInstancesResponse response = ec2.runInstances(runRequest);
String instanceId = response.instances().get(0).instanceId();

// 4. Tag it with a human-readable name
Tag tag = Tag.builder().key("Name").value(name).build();
CreateTagsRequest tagRequest = CreateTagsRequest.builder()
    .resources(instanceId)
    .tags(tag)
    .build();
try {
    ec2.createTags(tagRequest);
    System.out.printf("Started EC2 %s based on AMI %s",
                      instanceId, amiId);
} catch (Ec2Exception e) {
    System.err.println(e.awsErrorDetails().errorMessage());
    System.exit(1);
}
ec2.close();
Java — Enable CloudWatch monitoring on an existing instance
String instanceId = args[0];   // e.g. "i-1348636c"

Ec2Client ec2 = Ec2Client.builder()
    .region(Region.US_WEST_2)
    .build();

// MonitorInstances enables detailed 1-minute CloudWatch metrics
MonitorInstancesRequest request = MonitorInstancesRequest.builder()
    .instanceIds(instanceId)
    .build();

ec2.monitorInstances(request);
System.out.printf("Enabled monitoring for instance %s", instanceId);
ec2.close();
SDK Fluent Builder Pattern
Every AWS SDK request uses .builder() … .build() — immutable, type-safe, IDE-friendly. The client is created once per region and reused. All resource identifiers (instance IDs, ARNs, queue URLs) are returned as strongly-typed strings in the response object. SDK docs: 73,000+ classes — roughly the size of ten Java SE platforms.
