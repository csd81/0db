Map — (k1,v1) → list(k2,v2)
One record in, zero or more pairs out. Stateless, parallelisable.
Shuffle & Sort — group by key
Framework sorts all intermediate pairs so each Reduce sees one key at a time.
Reduce — (k2, list(v2)) → list(v2)
Merge all values for a key into one output. Produces final result.
Combiner — partial reduce on mapper
Pre-aggregates before network transfer. Same code as Reduce. Saves bandwidth.
Programming Model
map   (k1,v1)       → list(k2,v2)
reduce (k2, list(v2)) → list(v2)
WordCount — map emits (word,"1"); reduce sums counts
Distributed Grep — map emits matching lines; reduce = identity
Inverted Index — map emits (word, docID); reduce builds posting list
This demo — map emits (country, revenue); reduce sums → revenue by country
Inspiration: Lisp's map and fold. Restricting the model enables automatic parallelisation and fault-tolerance hidden from the programmer.
Implementation Details
Locality optimisation
Master schedules map tasks on the machine holding the GFS block. Most input is read from local disk — zero network bandwidth consumed. Move computation to data, not data to computation.
Backup tasks (straggler mitigation)
Near completion, master launches duplicate executions of remaining tasks. First to finish wins. Disabling this causes 44% slowdown on sort benchmark. Stragglers arise from bad disks, shared scheduling, CPU cache disabled by bugs.
Fault tolerance
Master pings workers periodically. Failed worker → re-execute all its map tasks (output on local disk, inaccessible). Killing 200/1746 workers mid-sort → only 5% slowdown. Re-execution is the primary fault-tolerance mechanism.

CPU-bound → I/O-bound
Old HPC: algorithms, simulations (MPI/OpenMP). Modern Big Data: datasets too large for any single machine. The bottleneck is reading data, not computing it.
3V Model — Volume · Velocity · Variety
Volume: orders of magnitude beyond one machine. Velocity: continuous ingestion speed. Variety: logs, images, JSON, sensor streams — not just rows.

Hadoop — 4 Components
Hadoop Common — shared utilities and libraries used by all modules
HDFS — high-performance distributed filesystem. Splits files into 128 MB blocks, replicates 3×. All Map input and Reduce output goes here. Locality optimisation: YARN schedules map tasks on the node holding the block.
YARN — cluster coordinator. Resource Manager (global scheduler) → Node Managers (per machine) → App Master per job negotiates containers. Clients submit jobs to Resource Manager.
MapReduce — batch processing engine running inside YARN containers. Implements the Map → Shuffle/Sort → Reduce pipeline. Java-native, accessible from other languages via Hadoop Streaming.
