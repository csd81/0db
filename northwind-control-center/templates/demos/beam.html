▶ Apache Beam Portable Pipeline Write once — run on any runner (Direct, Spark, Flink, Dataflow)

Apache Beam expresses data pipelines as a directed acyclic graph of PCollection objects flowing through PTransform stages — Read, Filter, Map, GroupByKey, Combine, Write — with no runner-specific code anywhere in the pipeline definition. The same Python or Java DAG runs unchanged on Direct Runner (laptop), Google Cloud Dataflow (serverless), Apache Spark, or Apache Flink by swapping a single configuration flag. This demo builds a WordCount pipeline against Northwind product descriptions and simulates what each runner does with the identical graph. Beam's write-once-run-anywhere abstraction is the reason Dataflow pipelines can be developed locally and deployed globally without code changes.

Beam Model
PCollection
Immutable distributed dataset — the data flowing between transforms. Can be bounded (batch) or unbounded (streaming).
PTransform
A data processing operation (Read, Filter, Map, GroupByKey, Combine, Write). Transforms PCollections into new PCollections.
Runner
The execution backend. Same Beam program runs on Direct, Spark, Flink, or Dataflow by changing one line: --runner=DataflowRunner
Windowing
For streaming: tumbling, sliding, session windows group unbounded data so GroupByKey + Combine can apply.
Portability
Beam SDK (Python/Java/Go) compiles to a portable format. Runners receive the same graph regardless of SDK language.
