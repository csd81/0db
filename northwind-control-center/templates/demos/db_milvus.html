Milvus

Vector Database · HNSW + IVF + DiskANN · Cloud-native · Billion-scale ANN search

Open-sourced in 2019 by Zilliz, Milvus is the largest open-source vector database by GitHub stars and the go-to choice for enterprise-scale similarity search. Built cloud-native from the ground up — separate coordinator nodes, worker nodes, and object storage — it scales to billions of vectors while maintaining sub-100ms query latency. Milvus integrates FAISS (Facebook AI Similarity Search) as a core indexing library and supports multiple ANN algorithms: HNSW (graph-based), IVF (inverted file), DiskANN (disk-resident for memory efficiency), and SCANN (Google's approach). Used by Walmart for product search, Shopee for recommendations, and NVIDIA for its cuVS integration.

Data Hierarchy

Collection — top-level entity (like a table). Has a schema defining fields and their types. Partitioned for organizational/performance purposes.

Partition — logical sub-division of a collection (e.g., by category or date). Enables partition-level search to reduce search scope.

Entity (Row) — a single record with a primary key, vector field(s), and optional scalar fields (integers, strings, booleans).

Dense Vector — float32 arrays (typically 128-4096 dims). The main search field. Multiple vector fields allowed per collection.

Sparse Vector — token→weight mappings for BM25/keyword-style hybrid search alongside dense vectors.

Segments — internal storage unit. Growing segments (in memory) are sealed and persisted to object storage when full. Index built on sealed segments.

Schema Definition 
from pymilvus import MilvusClient, DataType

client = MilvusClient("http://localhost:19530")

# Define schema
schema = MilvusClient.create_schema(auto_id=False)
schema.add_field("id",          DataType.INT64,        is_primary=True)
schema.add_field("product_id",  DataType.INT64)
schema.add_field("title",       DataType.VARCHAR, max_length=512)
schema.add_field("category",    DataType.VARCHAR, max_length=64)
schema.add_field("price",       DataType.FLOAT)
schema.add_field("embedding",   DataType.FLOAT_VECTOR, dim=1536)
# Sparse for hybrid search
schema.add_field("sparse_vec",  DataType.SPARSE_FLOAT_VECTOR)

# Index parameters
index_params = client.prepare_index_params()
index_params.add_index(
    field_name="embedding",
    index_type="HNSW",
    metric_type="COSINE",
    params={"M": 16, "efConstruction": 200}
)
index_params.add_index(
    field_name="sparse_vec",
    index_type="SPARSE_INVERTED_INDEX",
    metric_type="BM25"
)

client.create_collection(
    collection_name="products",
    schema=schema,
    index_params=index_params
)
Python SDK Primer 
from pymilvus import MilvusClient
import numpy as np

client = MilvusClient("http://localhost:19530")

# 1. Insert entities
data = [
    {"id": 1, "product_id": 101, "title": "iPhone 15 Pro",
     "category": "electronics", "price": 999.0,
     "embedding": np.random.rand(1536).tolist()},
    {"id": 2, "product_id": 102, "title": "MacBook Pro M3",
     "category": "electronics", "price": 1999.0,
     "embedding": np.random.rand(1536).tolist()},
]
client.insert(collection_name="products", data=data)

# 2. Basic ANN search
query_vector = np.random.rand(1536).tolist()
results = client.search(
    collection_name="products",
    data=[query_vector],
    anns_field="embedding",
    limit=10,
    output_fields=["product_id", "title", "price"],
    search_params={"metric_type": "COSINE", "params": {"ef": 100}}
)
for hit in results[0]:
    print(f"id={hit['id']}, title={hit['entity']['title']}, score={hit['distance']:.4f}")

# 3. Filtered ANN search (pre-filtering)
results = client.search(
    collection_name="products",
    data=[query_vector],
    anns_field="embedding",
    filter="category == 'electronics' and price < 1500",
    limit=5,
    output_fields=["title", "price"],
    search_params={"metric_type": "COSINE", "params": {"ef": 200}}
)

# 4. Hybrid search (dense + sparse BM25)
from pymilvus import AnnSearchRequest, RRFRanker

dense_req = AnnSearchRequest(
    data=[query_vector],
    anns_field="embedding",
    param={"metric_type": "COSINE", "params": {"ef": 100}},
    limit=10
)
sparse_req = AnnSearchRequest(
    data=[{"apple": 0.8, "macbook": 1.2}],  # BM25 tokens
    anns_field="sparse_vec",
    param={"metric_type": "BM25"},
    limit=10
)
results = client.hybrid_search(
    collection_name="products",
    reqs=[dense_req, sparse_req],
    ranker=RRFRanker(k=60),  # Reciprocal Rank Fusion
    limit=10
)

# 5. Query (scalar filter, no vector)
entities = client.query(
    collection_name="products",
    filter="category == 'electronics' and price > 500",
    output_fields=["id", "title", "price"],
    limit=100
)

# 6. Collection stats
client.get_collection_stats("products")
Killer Use Cases

RAG (Retrieval-Augmented Generation) — store document embeddings in Milvus, retrieve the top-K most relevant chunks for LLM context injection. The backbone of enterprise AI document search.

E-commerce Semantic Search (Walmart) — customers searching "cozy winter jacket" semantically match "fleece parka" even without keyword overlap. Milvus handles billion-product catalogs at Walmart scale.

Recommendation Systems (Shopee) — user behavior embeddings (click sequences, purchase history) used to find similar users/items. Two-tower model outputs go into Milvus for online serving.

Duplicate Detection at Scale — media companies embed images/videos and search for near-duplicates (perceptual hashing → vector search → 99.9% duplicate pairs found instantly at billion scale).

Cloud-Native Components

Root Coordinator — manages collection metadata (schemas, partitions, segments). Stores metadata in etcd.

Data Coordinator + DataNode — manages segment lifecycle: growing → sealed → indexed. DataNodes write to object storage (S3/MinIO).

Query Coordinator + QueryNode — manages query routing and segment loading. QueryNodes load segments from object storage into memory on demand.

Index Coordinator + IndexNode — builds ANN indexes (HNSW, IVF) on sealed segments asynchronously. Index stored in object storage.

Proxy — stateless load balancer. Routes SDK requests to appropriate coordinator/node.

Write Path & WAL

Message Queue — Milvus uses Pulsar or Kafka as a Write-Ahead Log (WAL). Every insert/delete is published to the log before being applied.

Growing Segments — new entities go into in-memory growing segments (binlogs). Multiple DataNodes subscribe to WAL partitions.

Sealed Segments — when a growing segment reaches the threshold (default 512MB), it's sealed and persisted to object storage as immutable binlog files.

Index Building — IndexNode reads sealed segments from object storage and builds ANN indexes asynchronously. Built indexes stored back to object storage. QueryNodes load indexes on demand.
