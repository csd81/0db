Cloud Storage
Object store — immutable blobs in buckets.
4 classes: Standard / Nearline / Coldline / Archive. REST API only, no mount. 11 nines durability.
Cloud SQL / Spanner
Managed relational databases.
Cloud SQL: MySQL / PostgreSQL / SQL Server, zonal. Cloud Spanner: global, 99.999% SLA, distributed ACID transactions.
Cloud Bigtable
Wide-column NoSQL — petabyte scale.
Sub-10ms latency. Row key + column families + timestamped cells. Ideal for IoT, time-series, financial, graph data.
Firestore / Memorystore
Document NoSQL & managed cache.
Firestore: serverless, ACID transactions, live sync (formerly Datastore). Memorystore: managed Redis / Memcached.
Cloud Storage — 4 Storage Classes
ClassAccess frequencyMin. durationRetrieval fee
Standard Frequent / hot none none
Nearline ~ once / month 30 days yes
Coldline ~ once / quarter 90 days yes
Archive ~ once / year 365 days yes (highest)
All four classes live in the same API — you only change the storage class flag on the bucket or individual object. Lifecycle rules can automatically migrate objects from Standard → Nearline → Coldline → Archive as they age, cutting costs without manual intervention.
Cloud Bigtable — Data Model
Row key │ cf1:col_a │ cf1:col_b │ cf2:col_x ──────────┼─────────────┼─────────────┼───────────── sensor#01 │ t3: 22.1° │ t3: 48% │ t2: OK │ t2: 21.8° │ │ sensor#02 │ t3: 19.3° │ t3: 51% │ t3: WARN
Row key — single index; lexicographic scan. Design it to spread load evenly.
Column family — group of related columns declared at table creation; stored together on disk.
Column qualifier — dynamic; unlimited per row; only populated cells stored (sparse).
Timestamps — multiple versioned values per cell; older versions auto-expired by GC policy.
Best for: IoT telemetry, financial tick data, time-series, graph adjacency. Scales to petabytes with sub-10ms reads.
