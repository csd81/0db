Kafka vs Pub/Sub
Kafka (log model)
Messages stored on disk forever (or by retention policy).
Each consumer group tracks its own offset.
New consumer groups can replay from offset 0.
Message never deleted on acknowledgement.
Pub/Sub (ephemeral)
Message deleted once all subscribers acknowledge.
No replay — if a subscriber misses it, it's gone.
Lower storage cost; simpler delivery semantics.
Consumer lag = log_size − offset
Zero lag = consumer is caught up.
High lag = consumer is falling behind the producer.

Three speeds in this demo:
· Analytics: reads every message (fast)
· Search: reads every 2nd message (medium)
· Archiver: reads every 4th message (slow)
Use Kafka when:
  • Multiple teams consume same events
  • Event replay / audit trail needed
  • High-throughput streaming pipelines
