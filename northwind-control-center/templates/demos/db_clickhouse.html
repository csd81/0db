ClickHouse

Columnar OLAP · MergeTree · 100B+ rows/sec · Real-time analytics

ClickHouse was open-sourced by Yandex in 2016 after being built internally to power Yandex Metrica — the second-largest web analytics service in the world, processing trillions of rows. It is consistently the fastest columnar database for analytical queries, achieving scan rates that make BigQuery and Redshift look slow. Cloudflare runs ClickHouse to analyze billions of DNS queries per day in real time. Uber, eBay, and ByteDance use it for observability and user analytics at scale.

Why ClickHouse Wins on Speed
  • Column files: only reads the columns a query touches
  • SIMD vectorized execution (AVX-512 on modern CPUs)
  • Sparse primary index: skips whole granules (8192 rows)
  • Skip indexes (bloom filter, min-max) per column
  • LZ4 decompression is faster than memory bandwidth
  • Single-threaded I/O design avoids lock contention
  • Materialized views + AggregatingMergeTree pre-aggregate
MergeTree Storage Layout 
Table: page_views
Columns: user_id, page_url, ts, country, duration

On-disk per part (immutable data part):
  user_id.bin     ← compressed column data
  user_id.mrk3    ← mark file (granule offsets)
  page_url.bin
  page_url.mrk3
  ts.bin
  ts.mrk3
  ...
  primary.idx     ← sparse index (every 8192 rows)
                    stores: (ts, user_id) per granule
  skp_idx_url.idx ← skip index (bloom filter on url)
  skp_idx_url.mrk3

Query: SELECT COUNT() FROM page_views
       WHERE country='US' AND ts > now() - INTERVAL 1 DAY

Execution:
  1. Primary index: skip granules outside time range
  2. Skip index: bloom filter skips granules w/o 'US'
  3. Decompress country + ts columns only
  4. Count in SIMD vectorized loop
MergeTree Table Definition 
-- Core MergeTree table
CREATE TABLE page_views (
  ts         DateTime,
  user_id    UInt64,
  page_url   LowCardinality(String),  -- dictionary encode
  country    LowCardinality(String),
  duration   UInt16,
  is_bounce  UInt8
)
ENGINE = MergeTree()
PARTITION BY toYYYYMM(ts)   -- monthly partitions
ORDER BY (country, ts)       -- sparse index order
TTL ts + INTERVAL 90 DAY;   -- auto-delete old data

-- Bloom filter skip index on page_url
ALTER TABLE page_views
  ADD INDEX idx_url page_url
  TYPE bloom_filter(0.01)
  GRANULARITY 4;
MergeTree Family
MergeTreeBase engine. Immutable parts merged in background. Best general-purpose.
ReplacingMergeTreeDe-duplicates rows with same ORDER BY key during merge. Last-write-wins.
AggregatingMergeTreePre-aggregated states (e.g., uniqState, sumState). Pairs with materialized views for pre-computation.
CollapsingMergeTreeCDC-style: sign column (+1/-1) collapses old versions during merge. Good for mutable state.
ReplicatedMergeTreeAny of the above with ZooKeeper/ClickHouse Keeper for replication.
Array Functions & SAMPLE 
-- Array join: unnest an array column
SELECT user_id, tag
FROM events
ARRAY JOIN tags AS tag
WHERE ts > now() - INTERVAL 7 DAY;

-- arrayMap / arrayFilter on inline arrays
SELECT
  arrayMap(x -> x * 2, [1, 2, 3, 4]) AS doubled,
  arrayFilter(x -> x > 50, scores) AS high_scores
FROM students;

-- SAMPLE: query 1% of data (approximate)
SELECT page_url, count() * 100 AS approx_views
FROM page_views SAMPLE 0.01
GROUP BY page_url ORDER BY approx_views DESC LIMIT 10;

-- WITH ROLLUP / WITH TOTALS
SELECT country, city, sum(revenue)
FROM sales
GROUP BY country, city WITH ROLLUP;
Materialized Views 
-- Source table (raw events)
CREATE TABLE raw_events (
  ts        DateTime,
  user_id   UInt64,
  event     String,
  country   String
) ENGINE = MergeTree() ORDER BY ts;

-- Pre-aggregated summary (AggregatingMergeTree)
CREATE TABLE hourly_summary (
  hour    DateTime,
  country LowCardinality(String),
  users   AggregateFunction(uniq, UInt64),
  events  AggregateFunction(count)
) ENGINE = AggregatingMergeTree()
  PARTITION BY toYYYYMM(hour)
  ORDER BY (country, hour);

-- Materialized view: auto-populates on insert
CREATE MATERIALIZED VIEW mv_hourly
TO hourly_summary AS
SELECT toStartOfHour(ts) AS hour,
  country,
  uniqState(user_id) AS users,
  countState() AS events
FROM raw_events GROUP BY hour, country;

-- Query from summary (fast)
SELECT hour, country,
  uniqMerge(users), countMerge(events)
FROM hourly_summary
WHERE hour >= now() - INTERVAL 24 HOUR
GROUP BY hour, country;

Parts & Merges

Data parts: Each INSERT creates a new immutable data part on disk. Parts contain sorted, compressed column files. Background merge threads continuously merge small parts into larger ones.

Sparse primary index: ClickHouse doesn't index every row — it indexes every 8192nd row (a granule). Queries skip granules by consulting the primary index, then read only relevant granules from disk.

Replication: ReplicatedMergeTree tables use ClickHouse Keeper (ZooKeeper-compatible) to coordinate. Each shard replica independently verifies and applies parts from ZK-tracked log.
