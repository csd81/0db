Microsoft SQL Server

Relational OLTP · T-SQL · Always On AG · Dominant in .NET/Windows enterprise stacks

Launched in 1989 as a joint Microsoft/Sybase product, SQL Server has become the second-largest commercial RDBMS globally and the undisputed king of Windows/.NET enterprise environments. Its Transact-SQL (T-SQL) dialect, tight integration with Azure, Visual Studio, and Active Directory, and battle-tested Always On Availability Groups make it the default choice for ERP systems (Microsoft Dynamics), ASP.NET backends, and Fortune 500 analytics. SQL Server 2022 introduced ledger tables, Azure Synapse Link, and S3-compatible object storage integration.

Core Abstractions

Pages (8KB) — base I/O unit. Each page stores rows for a heap or index. Extents (8 pages) are the allocation unit.

Filegroups — logical groups of data files. OLTP on PRIMARY, archives on SECONDARY. Read-only filegroups for historical data.

Clustered Index — determines physical row order. Every table should have one (leaf pages = data pages). Without it, you get a heap.

Columnstore Index (2012+) — batch-mode columnar index enabling 10-100x analytics speedup on OLTP tables. No schema change needed.

In-Memory OLTP (Hekaton) — memory-optimized tables with lock-free MVCC, compiled stored procedures, 30x OLTP speedup.

Temporal Tables (2016+) — system-versioned history table automatically maintained. Query AS OF a past time.

Schema Example 
-- Temporal table with history
CREATE TABLE dbo.Employees (
    EmployeeID   INT          IDENTITY PRIMARY KEY,
    Name         NVARCHAR(100),
    Department   NVARCHAR(50),
    Salary       DECIMAL(10,2),
    ValidFrom    DATETIME2 GENERATED ALWAYS AS ROW START,
    ValidTo      DATETIME2 GENERATED ALWAYS AS ROW END,
    PERIOD FOR SYSTEM_TIME (ValidFrom, ValidTo)
)
WITH (SYSTEM_VERSIONING = ON
    (HISTORY_TABLE = dbo.EmployeesHistory));

-- Columnstore index on OLTP table
CREATE NONCLUSTERED COLUMNSTORE INDEX
    cci_orders ON dbo.Orders
    (OrderDate, CustomerID, Total, Status)
    WHERE Status = 'COMPLETED';

-- Query temporal data
SELECT Name, Salary
FROM dbo.Employees
FOR SYSTEM_TIME AS OF '2023-01-01';
T-SQL Primer 
-- 1. Window functions with OFFSET-FETCH pagination
SELECT TOP (10) WITH TIES
    CustomerID, Name,
    SUM(Total) OVER (PARTITION BY CustomerID) AS lifetime_value,
    ROW_NUMBER() OVER (ORDER BY SUM(Total) DESC) AS rank
FROM Orders o JOIN Customers c USING (CustomerID)
GROUP BY CustomerID, Name
ORDER BY lifetime_value DESC
OFFSET 0 ROWS FETCH NEXT 10 ROWS ONLY;

-- 2. CTE with recursion (org chart)
WITH OrgChart AS (
    SELECT EmployeeID, ManagerID, Name, 0 AS Level
    FROM Employees WHERE ManagerID IS NULL
    UNION ALL
    SELECT e.EmployeeID, e.ManagerID, e.Name, oc.Level + 1
    FROM Employees e JOIN OrgChart oc ON e.ManagerID = oc.EmployeeID
)
SELECT REPLICATE('  ', Level) + Name AS HierarchyName FROM OrgChart;

-- 3. MERGE (upsert)
MERGE dbo.Inventory AS tgt
USING dbo.Staging AS src ON tgt.ProductID = src.ProductID
WHEN MATCHED THEN
    UPDATE SET tgt.Qty = src.Qty, tgt.Updated = GETDATE()
WHEN NOT MATCHED THEN
    INSERT (ProductID, Qty, Updated)
    VALUES (src.ProductID, src.Qty, GETDATE());

-- 4. JSON support (2016+)
SELECT JSON_VALUE(payload, '$.userId') AS user_id,
       JSON_QUERY(payload, '$.items') AS items
FROM Events
WHERE ISJSON(payload) = 1
  AND JSON_VALUE(payload, '$.type') = 'purchase';

-- 5. OPENJSON — expand JSON array to rows
SELECT j.order_id, j.product, j.qty
FROM Events e
CROSS APPLY OPENJSON(e.payload, '$.items')
    WITH (order_id INT, product NVARCHAR(100), qty INT) j;

-- 6. DBCC CHECKDB — integrity check
DBCC CHECKDB ('Northwind') WITH NO_INFOMSGS, ALL_ERRORMSGS;

Storage Engine

Buffer Pool — 8KB page cache in RAM. Dirty pages written to disk by lazywriter/checkpoint. Key tuning: max server memory.

Write-Ahead Log — transaction log (.ldf) is sequentially written. COMMIT returns after log flush. Checkpoint writes dirty data pages.

Row vs. Page Compression — ROW compression eliminates fixed-width overhead; PAGE adds dictionary compression across the page.

Extent Management — Mixed extents (multiple objects) vs. Uniform extents (single object). GAM, SGAM, IAM pages track allocation.

Always On AG

Always On Availability Groups (2012+) provide enterprise HA/DR: up to 8 secondary replicas, synchronous or async commit, readable secondaries for analytics offload, and automatic failover in <30 seconds.
