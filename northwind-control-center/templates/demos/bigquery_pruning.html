Columnar Storage
Row store (OLTP)
All columns of a row stored together.
Fast for INSERT/UPDATE (write one row).
Slow for analytics (must read unused columns).

Columnar store (OLAP)
Each column stored as a separate file stripe.
Fast for analytics (read only needed columns).
Excellent compression (same-type values).
Used by: BigQuery, Parquet, ORC, DuckDB.

Example: Query 2 of 15 columns on 10 TB table.
Columnar reads: 2/15 × 10 TB = 1.33 TB
Row store reads: 10 TB
Savings: 87%
Pruning Best Practices
❌ Never use SELECT *
✓ Always list required columns
❌ WHERE on non-partition col
✓ Filter on partition column first
❌ Cluster key not in WHERE
✓ Use cluster key in filter
❌ OR across partition col
✓ Use BETWEEN or IN with known values
Partitioning = coarse pruning (whole partitions skipped).
Clustering = fine pruning (blocks within partition skipped).
