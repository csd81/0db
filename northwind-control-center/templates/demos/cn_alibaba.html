⚠ The Breaking Point: 1.4B People + Singles' Day

Alibaba's shopping festival "Double 11" (November 11th) generates more transactions in one day than Amazon, eBay, and Walmart combined during their entire holiday season. The 2013 peak — 583,000 orders per second — exposed a catastrophic truth: Western enterprise databases were built for Western-scale traffic.

Oracle Row Locking

When millions of users simultaneously try to buy the last 1,000 units of an iPhone, every transaction must lock the inventory row to decrement the count. Oracle's lock manager became the serialization bottleneck — the database queue grew faster than it could drain, causing timeouts and cascade failures.

EMC Storage Latency

EMC SAN (Storage Area Network) arrays, while reliable, introduced network I/O latency between compute and storage. Under Singles' Day write storms, SAN queue depth saturated — a single disk write could take 200–400ms instead of <1ms, cascading across the OLTP layer.

The De-IOE Mandate (2009): Alibaba CTO Wang Jian initiated a company-wide campaign to eliminate all IBM minicomputers, Oracle databases, and EMC storage from their stack. The trigger: Oracle's licensing cost was growing faster than Alibaba's revenue, and no vendor support plan could handle Singles' Day load. The only solution was to build from scratch.

✦ OceanBase: Distributed Relational Database with Paxos Consensus

OceanBase is a shared-nothing distributed RDBMS — no centralized storage, no single point of failure. Transaction coordination uses the Paxos consensus algorithm to ensure that a write is committed only after a majority of replicas acknowledge it. This guarantees ACID semantics even if entire server racks lose power mid-transaction.

-- OceanBase: How "buy last iPhone" works at 583K orders/second
-- Row partitioned across N shards by product_id consistent hash
-- Each shard has 3 Paxos replicas (1 leader + 2 followers)

-- Step 1: Client routes to OBProxy → finds leader shard for product_id=1234
BEGIN;
  -- Step 2: SELECT with LOCK — acquires row lock on THIS shard's leader only
  SELECT units_in_stock INTO @stock
  FROM inventory
  WHERE product_id = 1234
  FOR UPDATE;                      -- Paxos lock, not global Oracle lock

  IF @stock > 0 THEN
    UPDATE inventory
       SET units_in_stock = units_in_stock - 1
     WHERE product_id = 1234;

    INSERT INTO orders (user_id, product_id, amount, created_at)
    VALUES (@user, 1234, @price, NOW());

    -- Paxos 2-phase commit: leader proposes → majority of 3 replicas ack
    -- COMMIT only succeeds when quorum confirms — survives rack failure
    COMMIT;
  ELSE
    ROLLBACK;  -- stock is 0 → clean rejection, no phantom orders
  END IF;

-- 583,000 such transactions per second, across thousands of shards
-- Each shard handles its slice in parallel — no global lock contention

✦ PolarDB: Cloud-Native Database (Alibaba's Aurora)

Alibaba Cloud's PolarDB follows the same compute-storage separation principle as AWS Aurora and Snowflake, but is specifically optimized for extreme OLTP concurrency. A single PolarDB instance supports up to 128 read replicas sharing the same underlying storage — all reading the same log stream without data copying. Write scale-out uses a shared NVMe-oF storage pool with 1M+ IOPS.

Traditional MySQL Master-Slave
  • Writes go to master only (bottleneck)
  • Replication lag: seconds under write storm
  • Read replicas get stale data
  • Adding a replica = full data copy (hours)
  • Disk I/O couples compute to storage
PolarDB (Compute-Storage Split)
  • Writes to shared log (redo log streaming)
  • Replica lag: <10ms (log replay, not data copy)
  • 128 read nodes, all consistent via log
  • New replica = attach to shared storage (seconds)
  • NVMe-oF: 1M IOPS shared across all compute
The De-IOE Timeline
2009 — De-IOE campaign launched. Oracle removed from all non-financial systems first. MySQL + internal sharding middleware (TDDL) deployed.
2010 — OceanBase 0.1 internal release. First Paxos-based distributed SQL on commodity x86 hardware.
2013 — Singles' Day peak hits 583K orders/second. Oracle fully eliminated from transaction path. OceanBase handles the entire load.
2015 — Alibaba Cloud (Aliyun) launches PolarDB as a commercial service. De-IOE exported as cloud product.
2020 — OceanBase 3.0 sets new TPC-C world record: 707 million tpmC — surpassing Oracle and all Western competitors.
2021 — OceanBase open-sourced (Apache 2.0). China's distributed database architecture becomes available globally.

The Verdict

Alibaba's De-IOE campaign was the most consequential database architectural decision of the 2010s. Forced by scale that Western vendors couldn't address, Alibaba built OceanBase — and then beat Oracle on Oracle's own benchmark. The lesson: existential traffic pressure is the mother of database invention. What started as a survival necessity became a $3B+ cloud database business exported globally.
