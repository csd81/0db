⚠ The Breaking Point: The Browser Can't Draw This

In 2012, design tools were native apps (Sketch, Illustrator, Photoshop). Browsers rendered HTML/CSS — fine for documents, catastrophically slow for interactive vector graphics with hundreds of layered objects, real-time shadows, and bezier curves.

DOM Rendering Failure

Rendering a design file as DOM elements caused reflow/repaint cascades. Scrolling a 1000-layer Figma doc via HTML would peg a CPU at 100% and drop to ~4fps — unusable for professional design work.

Collaboration Race Conditions

Real-time multiplayer means simultaneous edits. "Last write wins" destroys work. Operational Transform (OT) — used by Google Docs — requires a central sequencer that serializes all edits, creating a bottleneck at scale.

✦ Solution Part 1: WebGL Canvas — Bypass the DOM Entirely

Figma founder Evan Wallace (who wrote the WebGL spec implementation in Chrome) realized that the browser's <canvas> element with WebGL gave direct GPU access — the same API used by games. By writing a custom scene graph renderer on top of WebGL, Figma could draw 10,000+ vector objects at 60fps by issuing GPU draw calls directly, bypassing every layer of browser layout.

GPU Batching: Instead of one draw call per object (slow), Figma's renderer batches objects with the same material (fill type, blend mode) into a single GPU buffer upload. A 500-object frame becomes ~8 draw calls. This is identical to how game engines achieve 60fps with complex scenes. 
// Figma-style WebGL scene renderer (simplified)

class SceneRenderer {
  constructor(canvas: HTMLCanvasElement) {
    this.gl = canvas.getContext('webgl2', { antialias: true, alpha: false });
    this.batcher = new DrawCallBatcher(this.gl);
  }

  renderFrame(nodes: SceneNode[], viewport: Viewport) {
    const gl = this.gl;
    gl.clear(gl.COLOR_BUFFER_BIT | gl.DEPTH_BUFFER_BIT);

    // Sort by material to minimize GPU state changes
    const batches = this.batcher.group(nodes, n => `${n.fillType}-${n.blendMode}`);

    for (const batch of batches) {
      // Upload all vertices for this batch in one VBO
      gl.bufferData(gl.ARRAY_BUFFER, batch.toVertexBuffer(), gl.DYNAMIC_DRAW);
      // Single draw call for all objects sharing material
      gl.drawElements(gl.TRIANGLES, batch.indexCount, gl.UNSIGNED_INT, 0);
    }
    // Result: 10,000 nodes → ~12 draw calls (vs 10,000 DOM repaints)
  }
}

✦ Solution Part 2: WebAssembly — Ship C++ to the Browser

Vector path operations (bezier intersection, boolean geometry, text layout) are computationally intensive. JavaScript — even V8-compiled — can't match native speed. Figma wrote the entire rendering engine in C++ with Skia (same library as Chrome and Android), then compiled it to WebAssembly using Emscripten. The WASM binary loads in the browser and executes at near-native speed.

✦ Solution Part 3: CRDTs — Conflict-Free Multiplayer

Figma's multiplayer synchronization uses Conflict-free Replicated Data Types (CRDTs) — a class of data structures where concurrent edits from different clients are always mathematically mergeable with no conflicts, requiring no central coordinator.

Operational Transform (Google Docs)

All operations go through a central sequencer that transforms them relative to concurrent ops. Fast for linear text; fails on tree-structured objects (design layers). Requires always-online server sync.

CRDT (Figma)

Each property of each node is independently versioned. Concurrent edits on different properties always merge cleanly. Works offline — sync on reconnect. Scales to hundreds of collaborators per file.

# Figma CRDT: Last-Write-Wins Element Register (LWW-Element)
# Each node property is a separate CRDT with a Lamport timestamp

from dataclasses import dataclass
from typing import Dict, Any

@dataclass
class LWWProperty:
    value: Any
    timestamp: int       # Lamport clock — monotonically increasing
    client_id: str       # tiebreaker: higher client_id wins

    def merge(self, other: 'LWWProperty') -> 'LWWProperty':
        # Deterministic winner: latest timestamp, then client_id tiebreak
        if other.timestamp > self.timestamp:
            return other
        elif other.timestamp == self.timestamp and other.client_id > self.client_id:
            return other
        return self

class FigmaNode:
    # Each node property is an independent LWW register
    props: Dict[str, LWWProperty] = {}

    def set_prop(self, key: str, value: Any, ts: int, client: str):
        incoming = LWWProperty(value, ts, client)
        if key in self.props:
            self.props[key] = self.props[key].merge(incoming)
        else:
            self.props[key] = incoming

# Client A moves rect to x=100, Client B moves same rect to x=200 — simultaneously
# A: timestamp=5, B: timestamp=5, B.client_id > A.client_id → B wins (x=200)

WASM Performance Gap: Figma's boolean path operations (union, subtract, intersect) run 8–15× faster than equivalent JavaScript. For complex path operations on large documents, WASM is the difference between instant and "spinning beach ball."

The Verdict

Figma proved that "browser apps" and "native apps" is a false dichotomy. With WebGL + WASM + CRDTs, you can build software in a browser that outperforms native apps in latency (collaboration) and matches them in rendering performance — while gaining cross-platform delivery for free. Adobe's $20B acquisition attempt (blocked by EU) was an acknowledgment that Figma had permanently disrupted the category.

Industry Legacy
WASM Everywhere

Figma proved WASM was production-ready for complex apps. Today: AutoCAD Web, Photoshop Web, Google Earth, video editors, and game engines all use WASM for compute-heavy workloads.

CRDT Adoption

Figma's success triggered a wave of CRDT adoption: Notion, Linear, Liveblocks, Yjs, and Automerge all bring CRDT-powered collaboration to new domains. OT is now considered legacy.

Design-as-Infrastructure

Figma's API, plugin system, and webhooks turned design files into living infrastructure artifacts — design tokens, component libraries, and handoff specs are now version-controlled data, not static exports.
