▶ VM Lifecycle Cold / Warm / Hot State Machine — IaaS Compute Engine

A virtual machine transitions between three billing states: stopped (cold — no compute cost but 30–90 s startup latency), idle and running (warm — full compute cost, instant availability), and actively serving requests (hot — full cost fully amortised across active requests). Cold starts require the hypervisor to allocate CPU and RAM, boot the OS, and initialise the application runtime before the first request can be served. The demo tracks per-second billing across state transitions and compares VM economics against Cloud Functions, which eliminates idle cost entirely by scaling to zero between requests. This cost model is why auto-scaling groups, scheduled scaling policies, and spot-instance strategies exist — and why 'just leave the VM running' is the most common source of unexpected cloud bills.

IaaS cost lesson
A VM in warm or hot state bills at $0.096/hr even with zero traffic. Over 30 days: $69/month per idle VM.

FaaS alternative — Cloud Functions:
First 2 million invocations/month free. Beyond that: $0.40 per million invocations. Zero cost when idle — no provisioned resources.
