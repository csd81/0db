▶ Bigtable Wide-Column Store Sparse versioned cells — Google Bigtable / HBase model

Google Bigtable addresses every cell by a three-part coordinate — row key, column family, and timestamp — and stores only cells that actually exist, making it efficient for billions of sparse rows where most columns are empty. Row key design is the primary performance lever: monotonically increasing keys like timestamps create write hotspots on a single tablet server, while reversed or salted keys distribute writes evenly across the cluster. The demo lets you browse a simulated Bigtable instance with row-key range scans, column-family filters, and multi-version cell history. This architecture powers Gmail, Google Analytics, and YouTube's view counter at planetary scale.
