Export → MySQL (.sql)
Lossy.
  • datetime2(7) → DATETIME(6): sub-microsecond precision lost
  • datetimeoffset → VARCHAR(34): timezone offset lost
  • uniqueidentifier → CHAR(36): no UUID enforcement
  • T-SQL procedures/triggers/functions: not exported
NULL preserved. PKs, FKs, simple defaults included.
Export → PostgreSQL (.sql)
Lossy (best cross-DB approximation).
  • datetime2(7) → TIMESTAMP(6): last sub-microsecond digit lost
  • datetimeoffset → TIMESTAMPTZ: normalised to UTC, offset lost
  • hierarchyid / sql_variant → TEXT
  • T-SQL procedures/triggers/functions: not exported
NULL preserved. PKs, FKs, identity columns, sequence resets included.
